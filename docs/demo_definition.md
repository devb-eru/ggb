# GGB 공개 데모 정의

## 1. 기준선

| 필드 | 값 |
| --- | --- |
| 상태 | ACCEPTED |
| 목표 마일스톤 | `M2_DEMO_COMPLETE` |
| 작업 기준일 | 2026-11-20 |
| 공개 수준 | Steam Next Fest와 일반 사용자에게 공개 가능한 품질 |
| 콘텐츠 종료점 | 2장 종료 |
| 예상 플레이 시간 | 2시간~3시간 30분 |

플레이 시간 범위는 플레이테스트에서 큰 이탈·진행 문제가 발견되지 않는 한 고정한다. 필수 진행과 모든 이벤트 진행의 세부 시간 분리는 실제 콘텐츠 계측 후 기록한다.

## 2. 지원 범위

| 항목 | 기준 |
| --- | --- |
| OS | Windows 10/11 64-bit |
| 언어 | 한국어, 영어 |
| 목표 해상도 | `1920 × 1080` 가로형 |
| 화면 모드 | 창 모드·전체 화면 지원 |
| 선택 기능 | 창 크기 변경·해상도 선택은 일정 여유가 있을 때 구현 |
| 주 입력 | 마우스 |
| 보조 입력 | `ESC` 등 일부 키보드 입력 |
| 음성 | 별도 음성 없음 |
| 필수 접근성 | 대화 기록 열람 |
| 임시 아트·사운드 | 데모에서 노출되는 필수 에셋의 20% 이하 |

임시 리소스 비율은 공개 데모에서 실제로 노출되는 필수 에셋 목록을 기준으로 계산한다.

```text
임시 리소스 비율
= 임시 상태인 노출 필수 에셋 수 / 전체 노출 필수 에셋 수 × 100
```

사용되지 않는 파일, 개발 도구용 파일, 중복 원본은 분모에서 제외한다. 에셋 목록에는 종류, 노출 장면, 임시 여부, 교체 담당자와 교체 목표일을 기록한다. 20% 제한은 데모에만 적용한다.

## 3. 저장 데이터 승계

- 정식판까지 저장 구조가 호환되면 데모 저장 데이터와 대화 기록을 이어서 사용한다.
- 구조가 바뀌어 안전한 마이그레이션을 보장할 수 없으면 하드 리셋한다.
- 저장 파일에 `save_schema_version`, 빌드 버전, 완료한 콘텐츠 경계를 기록한다.
- 지원 가능한 버전은 마이그레이션하고, 미지원 버전은 실행 전에 경고와 초기화 사유를 표시한다.
- 승계 여부는 정식판 릴리스 후보 빌드 전에 QA 시나리오로 검증한다.
- 대화 기록은 저장 파일에 포함하고 저장·불러오기 뒤에도 유지한다. 새 게임과 하드 리셋에서만 초기화한다.

## 4. Steam 공개 방식

별도 데모 상점 페이지를 만들지 않고 **기본 게임 Coming Soon 페이지의 데모 버튼**을 사용한다. 결정 근거는 [GGB-DEC-2026-0003](decisions/GGB-DEC-2026-0003_데모_상점_페이지_운영.md)에 기록한다.

이 방식은 찜하기와 홍보 유입을 기본 게임 페이지에 모으고, 한국어·영어 설명과 시각 자산의 이중 관리를 줄인다. Steam 데모 자체는 기본 게임과 연결된 별도 AppID·빌드로 운영한다.

Next Fest에서도 공개 범위는 2장까지 유지한다. 기본 게임은 출시 후보 수준으로 완성하되 전체 콘텐츠를 데모 AppID에서 공개하지 않는다. 근거는 [GGB-DEC-2026-0004](decisions/GGB-DEC-2026-0004_Next_Fest_데모_공개_범위.md)를 따른다.

## 5. 완료 조건

- [ ] 새 게임에서 2장 종료까지 필수 경로를 완료할 수 있다.
- [ ] 목표 플레이 시간 2시간~3시간 30분 범위가 대표 플레이테스트에서 확인된다.
- [ ] 한국어와 영어에서 진행 조건, 일회성 정보, UI가 동일하게 동작한다.
- [ ] 마우스 중심 조작과 모든 필수 키보드 입력이 화면에 안내된다.
- [ ] Windows 10/11 64-bit에서 설치, 실행, 저장, 불러오기, 종료가 정상 동작한다.
- [ ] 창 모드·전체 화면 전환과 `1920 × 1080` UI가 깨지지 않는다.
- [ ] 노출 필수 에셋 기준 임시 리소스 비율이 20% 이하다.
- [ ] 음성 없이도 모든 정보가 전달된다.
- [ ] 확인한 대사와 선택지를 대화 기록에서 순서대로 다시 읽을 수 있고 저장·불러오기 뒤에도 유지된다.
- [ ] 데모 종료 뒤 기본 게임 상점과 찜하기 동선이 표시된다.
- [ ] `P0`, `P1` 문제가 없고 알려진 `P2`는 영향과 우회 방법이 기록된다.

## 6. QA 기준

최소 검증 조합은 다음과 같다.

| 경로 | 한국어 | 영어 | 마우스 | 키보드 보조 | 저장·재개 |
| --- | --- | --- | --- | --- | --- |
| 필수 진행 | 필수 | 필수 | 필수 | 필수 | 필수 |
| 선택 이벤트 | 필수 | 필수 | 표본+핵심 | 표본+핵심 | 분기별 필수 |
| 데모 종료 | 필수 | 필수 | 필수 | 필수 | 필수 |

QA 시간은 `gatam`, `beru`의 일일 개인 공통 가용시간에서 배분한다. 수동 전수 검증만 반복하지 않고 체크포인트 세이브, 데이터 정적 검사, 핵심 회귀 경로를 함께 사용한다.

## 7. 선택 구현

창 크기 변경과 해상도 선택 기능은 출시 차단 조건이 아니다. 핵심 진행, 저장, 한·영, 기본 창 모드와 전체 화면 QA를 먼저 통과한 뒤 남은 일정으로 구현한다.
