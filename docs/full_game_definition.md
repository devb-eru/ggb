# GGB 완성본 정의

## 1. 콘텐츠 범위

| 항목 | 기준 |
| --- | --- |
| 전체 구간 | 프롤로그~4장·엔딩 |
| 필수 진행 | 약 3~5시간 |
| 모든 콘텐츠 | 6시간 이상 |
| 2장 이후 콘텐츠 완료 목표 | 2026-11-20 |
| Next Fest 시점 기본 게임 | 출시판과 거의 동일한 완성본 |
| 출시 형태 | 정식 버전 1.0 |
| 출시 시점 | Next Fest 종료 후 최소 72시간 안정화, 가장 빠른 목표 2027-03-05 03:00 KST |

`2026-11-20`은 프롤로그~4장·엔딩의 콘텐츠가 처음부터 끝까지 연결되는 **콘텐츠 완료선**이다. 이후 기간은 밸런스, 접근성, 번역, 자산 교체, 성능, 버그 수정과 Steam 심사 대응에 사용한다.

Next Fest 데모 공개 범위는 [GGB-DEC-2026-0004](decisions/GGB-DEC-2026-0004_Next_Fest_데모_공개_범위.md), 행사 후 정식 1.0 출시는 [GGB-DEC-2026-0005](decisions/GGB-DEC-2026-0005_Next_Fest_후_정식_1_0_출시.md)를 따른다.

## 2. 제품 기준

- Windows 10/11 64-bit를 지원한다.
- 한국어와 영어 텍스트를 지원한다. 현재 계획에는 음성이 없다.
- 마우스를 주 입력으로 사용하고 `ESC` 등 필요한 키보드 입력을 보조로 사용한다.
- 창 모드와 전체 화면을 지원한다.
- 목표 해상도는 `1920 × 1080` 가로형이다.
- 창 크기 변경과 해상도 선택은 일정 여유가 있을 때 구현한다.
- 표시된 대사와 선택지를 다시 읽을 수 있는 대화 기록을 필수 지원한다. 기록은 저장·불러오기 뒤에도 유지하고 새 게임·하드 리셋에서만 초기화한다.
- 정식 버전 1.0에는 임시 아트·사운드를 허용하지 않는다.
- Next Fest 시점의 기본 게임은 최종 출시 빌드와 콘텐츠·저장 구조가 거의 같아야 한다. 행사 데모는 같은 품질 기준을 사용하되 콘텐츠는 2장까지만 공개한다.

## 3. 데모와의 관계

| 항목 | 데모 | 완성본 |
| --- | --- | --- |
| 종료점 | 2장 | 4장·엔딩 |
| 예상 시간 | 2시간~3시간 30분 | 필수 3~5시간, 전체 6시간 이상 |
| 음성 | 없음 | 없음 |
| 저장 승계 | 구조 호환 시 승계, 그 외 하드 리셋 | 데모 저장 마이그레이션 또는 초기화 처리 |
| Steam 배포 | 기본 상점 페이지와 연결된 데모 AppID | 기본 게임 AppID |

## 4. 완료·출시 게이트

- [ ] 프롤로그~4장·엔딩이 처음부터 끝까지 진행된다.
- [ ] 필수 진행 3~5시간, 전체 콘텐츠 6시간 이상의 계측 근거가 있다.
- [ ] 한국어·영어에서 같은 진행과 결말에 도달한다.
- [ ] 음성 없이 모든 필수 정보와 감정선이 전달되고 대화 기록을 다시 열람할 수 있다.
- [ ] `1920 × 1080` 가로형의 창 모드와 전체 화면을 검증한다.
- [ ] Windows 10/11 64-bit 공개 빌드 QA를 통과한다.
- [ ] Steam 상점과 빌드 심사를 통과한다.
- [ ] Next Fest에서 발견된 출시 차단 문제를 해결한다.
- [ ] `P0`, `P1` 문제가 없다.
- [ ] 정식 버전 1.0의 가격과 출시 할인이 확정되어 있다.
- [ ] Next Fest 종료 뒤 72시간 이상 안정화하고 출시 차단 문제를 재검증한다.

## 5. 후속 확정·선택 구현

- 창 크기 변경과 해상도 선택은 일정 여유가 있을 때 구현한다.
- 가격, 출시 할인율과 고객 지원 채널은 별도로 확정한다.
