# GGB 결정 기록

이 폴더는 프로젝트의 중요한 기획·기술·운영 결정을 `GGB-DEC` ID로 보존한다.

## 등록 규칙

1. [결정 기록 템플릿](../templates/decision_record.md)을 복사한다.
2. ID는 `GGB-DEC-YYYY-NNNN` 형식으로 연도별 마지막 번호 다음 값을 발급한다.
3. 파일명은 `GGB-DEC-YYYY-NNNN_결정_제목.md` 형식으로 작성한다.
4. 제안 중에는 `PROPOSED`, 확정 후에는 `ACCEPTED`로 표시한다.
5. 기각한 제안도 삭제하지 않고 `REJECTED`와 이유를 남긴다.
6. 기존 결정을 대체할 때 새 ID를 발급하고 기존 기록을 `SUPERSEDED`로 바꾼다.
7. 관련 작업, 충돌·오류, 기획 문서, 구현과 검증 근거를 양방향으로 연결한다.
