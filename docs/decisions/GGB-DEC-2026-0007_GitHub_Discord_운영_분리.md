## 결정

- GitHub Issue·Project를 작업, 담당자, 상태, 마감의 정본으로 사용한다.
- Discord를 중요 알림, 정기 회의, 에셋 접수 화면으로 사용한다.
- 즉시 행동이 필요한 사건은 실시간 알림, 일반 변경은 일일 요약으로 보낸다.
- 정기 회의는 매주 금요일 22:00 KST에 전원이 참석하며 기본 길이는 60분이다.

## 이유

모든 Git 이벤트를 실시간으로 보내면 중요한 차단과 검토 요청이 묻힌다. GitHub에는 구조화된 상태를 남기고 Discord에는 팀이 바로 행동해야 하는 정보만 전달하는 편이 검색, 자동 집계와 알림 효율이 높다.
