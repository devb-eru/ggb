## 질문

별도 데모 상점 페이지를 만들 것인가, 기본 게임 상점의 데모 버튼만 사용할 것인가.

## 선택지

| 선택지 | 장점 | 비용·위험 |
| --- | --- | --- |
| A. 기본 게임 페이지의 데모 버튼 | 찜하기와 홍보 유입 집중, 한·영 설명과 자산 관리 단순 | 데모 자체 페이지의 독립 홍보·리뷰 활용이 제한됨 |
| B. 별도 데모 상점 페이지 | 데모 전용 설명·검색·커뮤니티 운영 가능 | 별도 트레일러·스크린샷·설명과 이중 현지화·심사 필요 |

## 결정

**A를 채택한다.** 기본 게임 Coming Soon 페이지를 기준으로 삼고 데모 AppID를 연결한다. 별도 데모 상점 페이지는 만들지 않는다.

## 선택 이유

현재는 Steam 계정, AppID, 기본 상점 초안이 모두 없는 단계다. 정식 출시가 Next Fest 직후에 예정되어 있으므로 데모만의 장기 운영보다 기본 게임 찜하기와 출시 유입을 한 페이지에 모으는 편이 유리하다. 한국어·영어 설명과 시각 자산의 중복 관리도 줄어든다.

## 영향과 후속 작업

- 데모는 여전히 기본 게임과 연결된 별도 AppID·depot·build로 만든다.
- 기본 게임 상점의 설명과 자산이 데모 공개 전에 준비되어야 한다.
- 데모 종료 화면은 기본 게임 페이지와 찜하기로 연결한다.
- 독립적인 데모 마케팅 필요가 커지면 새 `GGB-DEC`로 재검토한다.
