# GGB Steamworks·Next Fest·출시 준비

## 1. 책임과 현재 상태

Steamworks·릴리즈 책임자는 `beru`다.

| 항목 | 상태 | 다음 행동 | 내부 목표 |
| --- | --- | --- | --- |
| Steamworks 개발자 계정 | BLOCKED | 개인 명의 법적·은행·세금 서류 준비 | 2026-08-14 |
| 계약 주체 | DECIDED | 개인 명의 정보로 온보딩 준비 | 완료 |
| 법적·은행·세금 서류 | NOT_READY | 필요한 서류 목록 확인과 준비 | 2026-08-03 |
| Steam Direct 수수료 | NOT_STARTED | 기본 게임 앱 수수료 결제 | 2026-08-14 |
| 기본 게임 AppID | BLOCKED | 가입과 수수료 완료 후 생성 | 2026-08-14 |
| 데모 AppID | BLOCKED | 기본 게임 AppID에서 생성 | 2026-11-20 이전 |
| Coming Soon 상점 | NOT_STARTED | 한·영 설명과 실제 게임 자산 준비 | 2026-10-30 |
| 게임명·한 문장 소개 | NOT_READY | 한·영 후보 작성과 승인 | 2026-07-31 |
| 가격·출시 할인 | NOT_READY | 비교작 조사 후 승인 | 2026-12-18 |
| 최소·권장 사양 | NOT_READY | 기준 장비 선정과 성능 계측 | 2026-12-18 |
| Next Fest 등록 | BLOCKED | 공개 상점과 자격 확인 후 등록 | 2027-01-08 |
| 출시 후보 | NOT_STARTED | 정식 1.0 범위·가격 확정 | 2027-02-12 |

Steamworks 가입에는 계약 주체, 은행·세금 정보와 앱당 100 USD의 Steam Direct 수수료가 필요하다. 첫 출시에는 수수료 결제 뒤 30일 대기와 최소 2주의 Coming Soon 공개 기간이 있으므로 계정과 상점 작업을 콘텐츠 완성 뒤로 미루지 않는다. 자세한 현행 조건은 [Steamworks 가입 안내](https://partner.steamgames.com/doc/gettingstarted/onboarding), [출시 옵션](https://partner.steamgames.com/doc/store/types?l=english), [상점·빌드 심사](https://partner.steamgames.com/doc/store/review_process?language=english)를 기준으로 한다.

## 2. 상점과 데모 운영 결정

기본 게임 Coming Soon 페이지 하나를 홍보 기준점으로 사용하고, 여기에 데모 버튼을 연결한다. 별도 데모 상점 페이지는 만들지 않는다. 결정 기록은 [GGB-DEC-2026-0003](decisions/GGB-DEC-2026-0003_데모_상점_페이지_운영.md)을 따른다.

Steam 배포 구조는 다음과 같다.

```text
기본 게임 AppID / Coming Soon 상점
└─ 데모 AppID / depot / build
```

데모는 별도 AppID와 빌드를 사용하지만 고객이 보는 설명과 찜하기 유입은 기본 게임 상점으로 모은다. Steam의 데모 구성과 선택 가능한 별도 상점 페이지 조건은 [Steam 데모 문서](https://partner.steamgames.com/doc/store/application/demos)를 기준으로 한다.

## 3. 구현 순서

### 3.1 계정과 AppID

- [x] Steam 계약 주체를 개인으로 정한다.
- [ ] 동일 명의의 은행·세금 정보를 준비한다.
- [ ] Steamworks 가입과 Steam Direct 수수료 결제를 완료한다.
- [ ] 기본 게임 AppID와 데모 AppID를 생성한다.
- [ ] 민감한 정보는 Git·Discord·공개 문서에 저장하지 않는다.
- [ ] `beru`에게 필요한 최소 Steamworks 권한을 부여한다.

### 3.2 기본 게임 상점

- [ ] 공식명, 짧은 설명, 상세 설명을 한국어·영어로 작성한다.
- [ ] 실제 게임 플레이를 보여주는 캡슐, 스크린샷, 트레일러를 준비한다.
- [ ] 장르, 태그, 기능, 언어, Windows 요구 사항을 확정한다.
- [ ] `1920 × 1080` 가로형, 창 모드·전체 화면 지원을 표기하고 검증한다.
- [ ] 현재 별도 민감 콘텐츠 고지 대상이 없다는 판단을 Steam 콘텐츠 설문에서 다시 확인한다.
- [ ] 상점 심사를 요청하고 피드백을 반영한다.
- [ ] 2026-10-30까지 Coming Soon 페이지를 공개한다.

### 3.3 데모

- [ ] [공개 데모 정의](demo_definition.md)의 2장·2~3.5시간 기준을 충족한다.
- [ ] Windows depot과 데모 build를 구성한다.
- [ ] 기본 게임 상점에 데모를 연결한다.
- [ ] 저장 승계·마이그레이션·하드 리셋 조건을 검증한다.
- [ ] 소유자 계정이 아닌 일반 Steam 계정으로 설치·실행·업데이트를 확인한다.

### 3.4 완성본과 Next Fest

- [ ] 2026-11-20까지 프롤로그~4장·엔딩의 콘텐츠 완료선을 만든다.
- [ ] 2027-01-08 내부 목표일까지 Next Fest 등록을 마친다.
- [ ] 2027-01-22까지 프레스 프리뷰용 심사를 제출한다.
- [ ] 2027-02-05까지 행사 필수 항목 심사를 제출한다.
- [ ] 2027-02-12까지 출시 후보 빌드를 고정한다.
- [x] Next Fest 데모의 공개 범위를 2장까지로 확정한다.
- [ ] 행사 중 크래시, 진행 불가, 저장 손상과 주요 피드백을 추적한다.
- [ ] 행사 종료 후 최소 72시간 안정화하고 게이트를 통과하면 `beru`가 정식 1.0 출시를 승인한다.

[February 2027 Next Fest 공식 안내](https://partner.steamgames.com/doc/marketing/upcoming_events/nextfest/feb_2027)에 따르면 행사는 2027-02-22 10:00 PST부터 2027-03-01 10:00 PST까지이며, 등록 마감은 2027-01-10 23:59 PST다.

## 4. 출시 게이트

- [x] 출시 형태를 앞서 해보기가 아닌 정식 버전 1.0으로 확정했다.
- [ ] Coming Soon 기간과 Steam Direct 대기 조건을 충족한다.
- [ ] 상점 페이지와 출시 빌드가 Valve 심사를 통과했다.
- [ ] 프롤로그~4장·엔딩 전체 진행이 가능하다.
- [ ] 한국어·영어와 Windows 10/11 64-bit QA를 통과했다.
- [ ] 음성 없이 모든 필수 정보가 전달되고 창 모드·전체 화면이 동작한다.
- [ ] 데모 저장 데이터 처리 방침이 실제 빌드에서 검증되었다.
- [ ] `P0`, `P1` 문제가 없다.
- [ ] 가격, 출시 할인, 지원 채널, 환불·법적 고지가 확정되었다.
- [ ] Next Fest에서 발견된 출시 차단 문제가 해결되었다.

행사 종료 시각에 자동 출시하지 않는다. 2027-03-01 10:00 PST 종료 후 최소 72시간을 확보하므로 가장 빠른 출시 목표는 2027-03-05 03:00 KST다.
