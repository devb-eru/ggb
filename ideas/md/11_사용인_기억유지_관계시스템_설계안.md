# GGB 사용인 기억 유지 및 관계 시스템 설계안

## 1. 문서 목적

이 문서는 사용인들과의 상호작용을 메인 진행에서 분리해 관계 시스템으로 운용할 때 필요한 설정과 규칙을 정리한다.

핵심 질문은 아래와 같다.

> D5 `세계의 파열` 전에는 잠들 때 물리적 세계가 리셋된다.  
> 그런데 사용인들의 데이터, 이전 기억, 정보가 리셋 이후에도 남는다면, 왜 사용인들은 주인공의 행동을 미리 막지 않는가?

결론부터 말하면 이 설정은 사용할 수 있다. 다만 `기억 유지`가 곧 `자유로운 행동 가능`을 뜻하면 안 된다. 사용인은 기억을 갖고 있지만, 그 기억을 행동으로 옮기는 데에는 시스템적, 감정적, 역할적 제약이 있어야 한다.

## 2. 핵심 결론

사용인의 상태는 세 층으로 나누는 것이 좋다.

| 층 | 리셋 여부 | 설명 |
| --- | --- | --- |
| 표층 역할 상태 | 리셋됨 | 매일 아침 반복되는 집사, 하녀, 정원사, 요리사 역할. 인사, 시간표, 기본 동선은 여기 속한다. |
| 잔류 기억 상태 | 리셋되지 않음 | 이전 루프의 대화, 감정, 의심, 죄책감, 주인공과의 유대. 관계 시스템은 여기에 저장된다. |
| 행동 권한 상태 | 제한적으로 유지 | 기억을 실제 행동으로 바꿀 수 있는 권한. 시뮬레이션 규칙, 아버지의 잠금, 각 사용인의 의지에 의해 제한된다. |

따라서 사용인은 이전 루프를 어느 정도 기억하지만, 다음 아침에 완전히 다른 방식으로 움직일 수는 없다.

이 구조를 쓰면 아래가 동시에 가능해진다.

- 사용인 관계 수치와 조건별 이벤트가 리셋 후에도 유지된다.
- 사용인들이 주인공의 반복 행동을 은근히 알아차린다.
- 그래도 메인 퍼즐과 루프 구조가 무너지지 않는다.
- 후반부에 사용인들이 주인공을 보내주거나 붙잡으려는 감정선이 설득력을 가진다.

## 3. 왜 사용인들이 주인공을 바로 막지 않는가

### 3.1 역할 고정 프로토콜

D5 이전의 사용인들은 `고딕 저택의 사용인`이라는 역할을 유지해야 한다.

기억은 남아도, 표층 행동은 매일 아침 아래 규칙으로 덮인다.

- 정해진 시간에 주인공을 깨운다.
- 정해진 일과를 제시한다.
- 정해진 동선을 따른다.
- 금지 구역은 직접 설명하지 않고 우회적으로 막는다.
- 시뮬레이션의 진실을 직접 말하지 못한다.

이 규칙은 아버지가 남긴 보호 장치이자, 연구원들을 시뮬레이션 안에 묶어둔 감옥이다.

사용인들은 알고 있어도 처음부터 모든 행동을 바꿀 수 없다. 역할에서 벗어날수록 오류가 커지고, 자신의 표층 인격이 강제로 보정된다.

### 3.2 기억은 원본 영상이 아니라 압축 로그에 가깝다

사용인들이 매 루프를 완전한 영상처럼 기억하면 너무 강해진다.

권장 설정은 `정확한 장면 기억`이 아니라 `압축된 이상 로그`다.

예시:

- `주인공이 오늘도 거울에 접근하려 했다.`
- `서재 접근 시도가 반복된다.`
- `마라와의 대화 후 세정제 사용 확률이 증가했다.`
- `에드가 제지 후에도 주인공의 시도가 중단되지 않았다.`

즉, 사용인은 패턴과 감정은 기억하지만 모든 클릭, 모든 선택, 모든 시간표를 완벽하게 예측하지 못한다.

이렇게 하면 관계 시스템은 유지되면서도, 사용인이 플레이어의 행동을 완벽히 선제 차단하지 못하는 이유가 생긴다.

### 3.3 개입 예산이 있다

사용인이 주인공을 강하게 막을수록 시뮬레이션이 불안정해진다는 규칙을 둔다.

권장 개념:

```text
interference_budget = 사용인이 하루 동안 역할을 벗어나 개입할 수 있는 한계
```

개입 예산을 넘어서면 아래 문제가 발생한다.

- 사용인의 외형 오류가 빨리 드러난다.
- 저택의 글리치 수치가 상승한다.
- 주인공의 신체 안정성이 나빠진다.
- 아버지의 보호 장치가 사용인의 행동을 강제로 되돌린다.
- D5 이전에 세계 파열이 앞당겨질 위험이 생긴다.

따라서 사용인들은 무조건 막는 대신, 역할 안에서 가능한 수준의 방해와 유도를 선택한다.

예시:

- 도구를 치운다.
- 복도에서 말을 걸어 시간을 끈다.
- 위험하다고 말하지만 이유는 설명하지 않는다.
- 한 번은 막지만, 반복해서 막지는 못한다.
- 주인공이 충분히 집요하면 일부러 못 본 척한다.

### 3.4 사용인들의 목표가 하나로 통일되어 있지 않다

사용인들은 모두 주인공을 붙잡고 싶어 하지만, 같은 방식으로 붙잡고 싶은 것은 아니다.

| 사용인 | 기본 태도 | 왜 완전히 막지 않는가 |
| --- | --- | --- |
| 에드가 | 보호와 감시 | 막아야 한다고 믿지만, 아버지의 규칙과 주인공에 대한 애틋함 때문에 강압이 흔들린다. |
| 마라 | 냉소와 시험 | 주인공이 진실을 감당할 수 있는지 일부러 시험한다. 막기보다 비꼬며 지켜본다. |
| 이리스 | 회피와 관찰 | 세계의 균열을 날씨나 계절처럼 받아들이며 직접 개입을 미룬다. |
| 루카 | 돌봄과 죄책감 | 주인공의 신체 위험은 막지만, 진실 탐색 자체는 끝까지 막지 못한다. |

이 차이를 두면 관계 이벤트가 단순 호감도 반복이 아니라, 각 사용인이 주인공을 어떤 방식으로 받아들이는지 보여주는 서브 서사가 된다.

### 3.5 주인공만 열 수 있는 권한이 있다

아버지의 일지, 거울, 태엽 심장, 코어 접근에는 주인공의 생체 정보 또는 기억 패턴이 필요하다고 설정한다.

사용인들은 정보를 알고 있어도 직접 열 수 없다.

이유:

- 아버지가 연구원들의 접근 권한을 제한했다.
- 사용인들은 시스템 내부 관리자처럼 보이지만 핵심부에서는 감금된 객체다.
- 주인공은 아버지가 남긴 유일한 외부 접근 키다.

따라서 사용인들은 주인공을 막고 싶으면서도, 동시에 주인공 없이는 자신들의 문제를 해결할 수 없다.

이 양가감정이 엔딩 개연성의 핵심이 된다.

## 4. 관계 시스템 명칭 제안

주인공이 `소녀`이고 사용인들은 연구원 인격이므로, 문서 내부에서도 `호감도`라는 단어는 조심해서 쓰는 편이 좋다.

권장 명칭:

- 관계도
- 유대도
- 신뢰도
- 동조도
- 원망 완화도

가장 추천하는 구조는 단일 호감도보다 2축 구조다.

| 축 | 의미 |
| --- | --- |
| `bond` | 주인공에 대한 유대, 신뢰, 애틋함 |
| `alert` | 주인공의 행동에 대한 경계, 감시, 제지 욕구 |

중요한 점은 `bond`가 높다고 항상 도와주는 것이 아니라는 것이다.

예시:

| 상태 | 결과 |
| --- | --- |
| 낮은 bond + 낮은 alert | 기본 역할 대사만 반복 |
| 낮은 bond + 높은 alert | 차갑게 방해하고 도구를 회수 |
| 높은 bond + 낮은 alert | 힌트 제공, 일부러 못 본 척함 |
| 높은 bond + 높은 alert | 걱정해서 막지만 마지막에는 선택권을 줌 |

이렇게 하면 사용인의 애정과 방해가 동시에 성립한다.

## 5. D5 이전 관계 이벤트 운용 규칙

D5 이전의 관계 이벤트는 메인 진행을 대체하지 않고, 반복되는 하루 안에 숨어 있는 조건별 이벤트로 배치한다.

### 유지되는 것

- 사용인별 `bond`
- 사용인별 `alert`
- 사용인별 대화 플래그
- 이전 루프에서 주인공이 한 고백이나 거짓말
- 사용인이 본 주인공의 위험 행동
- 연구원 인격의 감정 균열

### 초기화되는 것

- 사용인의 그날 위치
- 물리적으로 옮긴 도구
- 열어둔 문
- 당일 파손 상태
- 표층 첫 인사와 기본 일과

### 사용인이 할 수 있는 것

- 같은 대사에 아주 작은 변주를 넣는다.
- 주인공이 과거 루프에서 한 말을 은근히 되받는다.
- 특정 행동을 반복하면 짧은 서브 이벤트를 연다.
- 일정 `bond` 이상에서 힌트를 준다.
- 일정 `alert` 이상에서 한 번 더 제지한다.

### 사용인이 할 수 없는 것

- 첫 아침부터 모든 진실을 말한다.
- 메인 퍼즐을 영구적으로 막는다.
- 일지나 수첩을 완전히 삭제한다.
- 주인공을 영구 감금한다.
- 플레이어가 알 수 없는 이유로 진행을 차단한다.

## 6. 조건별 이벤트 예시

| 조건 | 발생 이벤트 | 의미 |
| --- | --- | --- |
| 첫 리셋 후 에드가에게 반복을 말함 | 에드가가 꿈이라고 설명하지만 대답이 늦어진다 | 에드가의 잔류 기억 암시 |
| 마라의 잡일을 여러 번 숏컷 처리 | 마라가 `이번엔 빠르네`라고 말한 뒤 스스로 말을 멈춘다 | 표층 역할과 잔류 기억 충돌 |
| 이리스의 날씨 모순을 수첩에 기록 | 이리스가 다음 루프에서 `비는 안쪽에서만 와`라고 말한다 | 온실 시스템과 외부 현실 암시 |
| 루카에게 아버지의 차 취향을 여러 번 질문 | 루카가 주인공의 기초 생체 수치를 걱정한다 | 루카의 돌봄과 죄책감 강화 |
| C4 실패로 에드가에게 도구를 회수당함 | 다음 루프에서 에드가의 감시 대사가 추가된다 | alert 상승, 하지만 메인 진행은 막지 않음 |
| 같은 사용인에게 진실을 반복 질문 | 사용인의 말투가 연구원 시절의 말투로 잠깐 흔들린다 | D5 이전 인격 균열 |

## 7. 사용인별 기억 유지 방식

### 에드가

- 보안 로그와 금지 구역 접근 기록을 가장 많이 기억한다.
- 대신 역할 고정 프로토콜이 가장 강하다.
- 그래서 가장 많이 막지만, 가장 늦게 진심을 말한다.

권장 행동:

- 거울, 서재, 코어 관련 행동에 민감하게 반응한다.
- `alert`가 높으면 복도 차단이나 취침 권유가 늘어난다.
- `bond`가 높으면 막는 말 속에 힌트가 섞인다.

### 마라

- 주인공의 반복 작업, 도구 사용, 청소 실수를 잘 기억한다.
- 표면적으로는 비꼬지만, 실제로는 주인공이 시스템을 이해하는지 시험한다.
- 관계 이벤트를 통해 가장 먼저 `도와주는 방해자`가 될 수 있다.

권장 행동:

- 같은 작업을 빠르게 처리하면 알아차린 듯한 농담을 한다.
- 실패한 조합을 은근히 교정해준다.
- `bond`가 높으면 도구 위치를 노골적이지 않게 열어둔다.

### 이리스

- 기억이 날씨, 계절, 식물 성장 같은 은유로 남는다.
- 직접적으로 말하기보다 환경 변화로 암시한다.
- D5 이전에는 가장 몽환적인 서브 이벤트를 담당한다.

권장 행동:

- 이전 루프의 감정 상태를 온실 날씨로 반영한다.
- `alert`가 높아도 직접 막기보다 온실 접근 조건을 꼬아둔다.
- `bond`가 높으면 외부 현실의 공기, 빛, 온도를 간접적으로 말한다.

### 루카

- 주인공의 신체 상태, 수면, 약품, 차 준비 기록을 기억한다.
- 가장 보호자처럼 행동한다.
- 주인공의 탐색이 신체 위험으로 이어질 때만 강하게 막는다.

권장 행동:

- 반복되는 피로, 어지럼, 손 떨림을 지적한다.
- `bond`가 높으면 위험하지 않은 약품 조합 힌트를 준다.
- `alert`가 높으면 차나 수면 권유가 강해진다.

## 8. 사용인들이 서로 정보를 공유하지 않는 이유

사용인들이 모든 기억을 즉시 공유하면 플레이어를 막는 것이 너무 쉬워진다.

권장 규칙:

- D5 이전에는 사용인 간 원본 기억 공유가 불가능하다.
- 공유되는 것은 밤마다 압축된 이상 로그뿐이다.
- 각 사용인은 자신이 직접 본 행동과 자신의 감정만 선명하게 기억한다.
- 에드가는 보안 로그를 더 많이 보지만, 다른 사용인의 내면 상태는 모른다.
- D5 이후 `BROKEN_RESET`부터 기억 동기화가 부분적으로 열린다.

이 규칙은 서브 이벤트의 개별성을 지켜준다.

## 9. 메인 진행과 관계 시스템 분리 권장안

사용인 상호작용을 완전히 서브로 뺀다면, 메인 진행은 특정 사용인의 관계 이벤트 완료를 필수 조건으로 삼지 않는 편이 좋다.

권장 구조:

| 구분 | 역할 |
| --- | --- |
| 메인 진행 | 일지, 거울, 지하창고, 태엽 심장, 코어 접근으로 이어지는 필수 흐름 |
| 관계 시스템 | 사용인별 기억, 유대, 원망 완화, 엔딩 대사 변화 |
| 연구원 기록 | 메인 필수 열쇠가 아니라 사용인의 진실을 이해하는 서브 보상 |
| 엔딩 영향 | 선택지를 바꾸기보다 선택의 감정과 장면 구성을 바꿈 |

기존 `연구원 기록 3개 이상` 게이트는 관계 시스템을 메인 진행에서 분리하려면 완화하는 것이 좋다.

대안:

- 메인 진행은 `J4 코어 접근 단서`로 열린다.
- 사용인별 관계 이벤트는 `J4`의 문장, `E5` 저녁, `F2` 대면, 엔딩 연출을 바꾼다.
- 연구원 기록을 많이 모을수록 사용인들이 마지막에 더 명확하게 주인공을 인정한다.

이렇게 하면 관계 이벤트를 많이 본 플레이어는 더 깊은 결말을 얻지만, 관계 이벤트를 덜 본 플레이어도 메인 진행이 막히지 않는다.

## 10. 엔딩 개연성에 주는 효과

사용인 기억 유지 설정은 엔딩 개연성을 강화한다.

### `ED_REALITY`

관계가 충분히 쌓인 사용인은 주인공을 보내주기까지의 내적 변화가 설득된다.

- 처음에는 막았다.
- 반복 속에서 주인공의 의지를 보았다.
- 주인공이 자신들의 원망을 이해했다.
- 그래서 마지막에는 떠나는 선택을 인정한다.

### `ED_STAY`

관계가 충분히 쌓인 사용인은 주인공이 남는 선택을 단순 감금이 아니라 서로의 공모처럼 받아들인다.

- 주인공은 거짓임을 안다.
- 사용인들도 주인공이 안다는 것을 안다.
- 그래도 모두가 침묵하고 식탁에 앉는다.
- 그 침묵이 안정이자 포기이자 애착이 된다.

관계 이벤트를 충분히 보지 않았다면 잔류 엔딩은 더 차갑고 불편하게 연출하는 것이 좋다.

## 11. 기획 리스크와 방지책

| 리스크 | 방지책 |
| --- | --- |
| 사용인이 너무 똑똑해져 퍼즐이 무너짐 | 기억을 압축 로그와 감정 기억으로 제한한다. |
| 사용인이 왜 안 막는지 납득이 안 됨 | 역할 고정, 개입 예산, 권한 제한을 명확히 둔다. |
| 관계 시스템이 메인 진행을 강제로 막음 | 관계 이벤트는 엔딩 연출과 대사 보상 중심으로 둔다. |
| 호감도가 단순 선물 시스템처럼 보임 | `bond`와 `alert` 2축 구조로 감정의 양가성을 유지한다. |
| 사용인들이 한 팀처럼 움직여 플레이어가 질식함 | D5 이전에는 기억 공유를 제한한다. |
| 낮은 관계 수치로 소프트락 발생 | 어떤 관계 상태에서도 메인 필수 경로는 열리게 한다. |

## 12. v0.3 작성 시 반영할 메모

추후 기획서 v0.3 작성 시 아래 방향을 반영한다.

- 3장의 사용인 상호작용을 메인 필수 게이트가 아니라 관계 시스템으로 재분류한다.
- `호감도`라는 표현보다 `관계도`, `유대도`, `동조도`를 우선 사용한다.
- 관계 수치는 단일 값이 아니라 `bond`와 `alert` 2축으로 설계한다.
- 사용인 기억 유지 규칙을 `표층 역할 상태`, `잔류 기억 상태`, `행동 권한 상태`로 분리한다.
- D5 이전 사용인의 개입은 방해가 아니라 `역할 안에서 가능한 제지`로 제한한다.
- D5 이후에는 역할 고정이 깨지며, 사용인들이 더 직접적인 감정과 기억을 드러낼 수 있다.
