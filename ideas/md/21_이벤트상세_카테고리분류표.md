# GGB 이벤트 상세 카테고리 분류표

## 1. 문서 목적

본 문서는 현재까지 정리된 전체 이벤트를 기능별 카테고리로 나누고, 이후 카테고리별 상세 이벤트 시트를 순차 작성하기 위한 기준표다.

이벤트는 챕터 순서만으로 나누지 않는다. 같은 챕터 안에서도 `튜토리얼`, `루프 검증`, `퍼즐`, `관계`, `연출 전환`, `엔딩` 기능이 다르기 때문이다.

## 2. 이벤트 상세 작성 공통 양식

각 이벤트 상세 문서는 아래 항목을 기본으로 작성한다.

| 항목 | 설명 |
| --- | --- |
| 이벤트 ID | 흐름도에서 사용하는 고유 ID |
| 이벤트명 | 플레이어가 이해할 수 있는 사건명 |
| 카테고리 | 아래 분류표 기준 |
| 발생 위치 | 방 또는 공간 노드 |
| 발생 시간대 | 아침, 낮, 저녁, 밤, 루프 후, BROKEN_RESET 이후 등 |
| 선행 조건 | 필요한 플래그, 이전 이벤트, 아이템, 관계 상태 |
| 플레이어 목표 | 플레이어가 당장 이해해야 하는 행동 목표 |
| 핵심 상호작용 | 클릭, 조사, 대화, 아이템 사용, 조합, 시간 대기 등 |
| 성공 조건 | 이벤트 완료 판정 |
| 실패 / 정체 처리 | 오답, 시간 초과, 잘못된 순서, 리셋 필요 여부 |
| 힌트 단계 | 정체 시간 또는 반복 실패에 따른 힌트 |
| 월드 변경 | 오브젝트, NPC, 시간대, 수첩, 일지, 균열 단계 변화 |
| 감각 / 심리 연출 | 불안, 반복감, 촉감, 소리, 온도, 빛 연출 |
| 구현 메모 | Godot Scene, Flag, Inventory, Dialogue 연동 메모 |

## 4. 카테고리별 상세 작성 순서

권장 작성 순서는 아래와 같다.

1. 튜토리얼 / 일상 이벤트
2. 루프 / 영구 정보 / 숏컷 이벤트
3. 메인 퍼즐 이벤트
4. 정보 조사 / 일지 복원 이벤트
5. 공간 잠금 / 해금 / 동선 이벤트
6. 사용인 짧은 반응 이벤트
7. 사용인 핵심 관계 이벤트
8. 파열 / 전환 / 결산 연출 이벤트
9. 엔딩 이벤트
10. 공통 오브젝트 반응 이벤트

이 순서의 이유:

- 먼저 플레이어의 기본 조작과 하루 구조를 확정한다.
- 그다음 루프와 영구 정보 규칙을 확정해야 퍼즐 실패 처리를 일관되게 설계할 수 있다.
- 퍼즐을 먼저 구체화한 뒤, 일지와 공간 해금이 어떤 정보를 보상으로 줄지 맞춘다.
- 사용인 관계 이벤트는 메인 진행과 분리되어 있으므로, 메인 루프와 퍼즐이 정리된 뒤 상세화하는 편이 안전하다.
- 엔딩과 오브젝트 반응은 앞선 모든 상태 변수를 참조하므로 마지막에 세밀화한다.

## 6. 이벤트 ID 표기 규칙

| 표기 | 의미 |
| --- | --- |
| P | 프롤로그 일상 및 튜토리얼 이벤트 |
| A~B | 1챕터 루프 검증 및 시간표, 열세 번째 종 구간 |
| C~D | 2챕터 거울, 지하창고, 태엽 심장, 파열 구간 |
| E | 3챕터 BROKEN_RESET 이후 관계 이벤트 구간 |
| F | 4챕터 코어 접근, 아버지 기록, 연구원 대면 구간 |
| J | 일지 복원 단계 |
| ED | 엔딩 관련 이벤트 |
| SYS | 반복적으로 작동하는 시스템 이벤트 |

## 7. 중복 배치 규칙

일부 이벤트는 두 개 이상의 기능을 가진다.

예를 들어 `P2 창문 닦기`는 튜토리얼이면서 퍼즐의 기초 조작을 가르친다. 하지만 상세 작성 시에는 최초 기능을 기준으로 한 번만 자세히 작성하고, 다른 카테고리에서는 참조만 한다.

| 이벤트 | 대표 카테고리 | 참조 카테고리 |
| --- | --- | --- |
| P2 창문 닦기 | 튜토리얼 / 일상 | 메인 퍼즐, 오브젝트 반응 |
| P3 책 정리 | 튜토리얼 / 일상 | 정보 조사 / 일지 복원 |
| A1 수첩 표시 | 루프 / 영구 정보 | 정보 조사 |
| B3 열세 번째 종 | 메인 퍼즐 | 정보 조사, 루프 실패 |
| C4 검은 거울 | 메인 퍼즐 | 공간 해금, 파열 전조 |
| D4 태엽 심장 | 메인 퍼즐 | 파열 / 전환 |
| E3_4 에드가 코어 복원 | 사용인 핵심 관계 | 공간 해금 |
| F2 연구원 대면 | 파열 / 전환 / 결산 | 엔딩 |
