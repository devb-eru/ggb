# GGB v0.4 이벤트 상세 09: 엔딩

## 1. 설계 원칙

- 현실은 용기, 잔류는 안정이라는 단순 도덕 판정을 하지 않는다.
- 현실은 불확실하고 위험하며, 잔류는 거짓임을 알고 선택하는 삶이다.
- 관계 이벤트는 선택지의 존재를 막지 않고 작별의 해상도만 바꾼다.
- 주인공의 SUBJECT 권한은 연필·종이·필기음으로 표현한다.
- 사용인의 색은 최종 반응에서 분리·공존·소실 상태를 보여 주지만 선택 UI의 정답 표시는 하지 않는다.
- F0-E의 의향은 현재 마음의 기록이지 예약된 엔딩이 아니다.
- 이전 의향과 다른 최종 선택은 실패·변심 페널티가 아니라 현재 권한의 정상적인 사용이다.

### 1.1 핵심 관계 outcome의 엔딩 사용 범위

E3_1~E3_5의 `event_history.outcome_id`는 엔딩의 짧은 대사·소품·행동 순서만 바꾼다. 어느 outcome도 현실·잔류 선택지, 기상 성공, 루프 안정화 성공, 관계 단계, 기록 정보량을 바꾸지 않는다.

| 사건·outcome | 현실 엔딩 overlay | 잔류 엔딩 overlay |
| --- | --- | --- |
| E3_1 `original_attribution` | 마라 1이 책임자명이 남은 감사 사본을 저전력 보존함에 넣음 | 삭제 금지 목록에 자기 이름을 직접 추가 |
| E3_1 `protected_identifiers` | 취약 대상 식별자를 가린 사본과 원본 해시를 함께 보존 | 새 규칙에 개인 식별자 보호 절차를 작성 |
| E3_2 `external_truth` | 실제 외부 계절을 확인해 달라는 요청을 먼저 말함 | 투영보다 외부 센서값을 우선 표시 |
| E3_2 `shelter_projection` | 온실 투영을 끄는 스위치를 주인공에게 넘김 | 투영·외부값을 나란히 표시하고 언제든 끌 수 있게 함 |
| E3_3 `full_disclosure` | 캡슐 위험표를 가감 없이 수첩 마지막 장에 전송 | 모든 생명 유지 경고를 숨김 없이 공동 열람 |
| E3_3 `stabilize_first` | 안정화 완료 시각 뒤 위험표를 순서대로 전송 | 경고 전 장치 안정 상태를 먼저 확인하는 규칙을 둠 |
| E3_4 `responsibility_recorded` | 에드가의 감사 서명이 종료 로그 마지막 줄에 남음 | 운영 기록에서 에드가의 감금 결정을 삭제하지 않음 |
| E3_4 `authority_returned` | 빈 CUSTODIAN 선택 슬롯과 주인공 SUBJECT 서명을 보여 줌 | 취침·기상 질문의 최종 확인자를 주인공으로 고정 |
| E3_5 `merged` | 병합된 마라 2 인격이 하나의 목소리로 작별 | 돌아온 주석과 현재 기억을 한 인덱스로 유지 |
| E3_5 `separated` | 원본과 주석이 번갈아 짧게 작별 | 두 인덱스를 별도 보존하고 교차 링크 유지 |

이리스의 outcome overlay는 `iris_confession_state` 대사 뒤에 붙는다. outcome이 고백 상태를 만들거나 덮어쓰지 않는다. 마라 2의 `mara2_name_written`도 E3_5 outcome을 재계산하지 않는 별도 소품 overlay다.

### 1.2 게이트 금지와 fallback

```yaml
ending_outcome_overlay_router:
  reads:
    - event_history.E3_1.outcome_id
    - event_history.E3_2.outcome_id
    - event_history.E3_3.outcome_id
    - event_history.E3_4.outcome_id
    - event_history.E3_5.outcome_id
  applies_when:
    - matching_E3_complete
    - outcome_id_valid
  affects_only: [dialogue_variant, prop_overlay, action_order]
  forbidden_effects:
    - final_decision
    - choice_visibility
    - settlement_tier
    - researcher_record_count
    - relationship_changes
    - ending_success
  invalid_or_missing_outcome:
    show: generic_completed_reaction
    log: ENDING_OUTCOME_INTEGRITY_WARNING
    block_ending: false
```

- E3_4M은 E3_4 outcome을 만들지 않는다. 해당 경로는 에드가 미완료 반응을 사용한다.
- 미완료 사용인은 기존 미완료 반응을 사용하며 outcome을 추정하지 않는다.
- `all_servants_complete`는 다섯 E3 완료 플래그로만 계산한다. E3_4M과 outcome 종류는 ALL 조건에 영향을 주지 않는다.
- 엔딩 화면의 기본 포커스·버튼 크기·색·확인 횟수는 outcome과 무관하다.

## 2. EDC 최종 선택

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | `H0_CORE_CHAMBER`, F3 완료 뒤 |
| 선행 조건 | `F3_complete`, `final_sleep_lock=true`, `subject_authority_restored`, `final_decision == unset` |
| 목표 | 현실 기상 또는 안정화 루프 복원 선택 |
| 상호작용 | F3 균형 요약 재열람, 선택 문장 자필 확인, 최종 실행 |
| 성공·실패 | 취소하여 F3 조사로 돌아갈 수 있음. 실행 후 확정 |
| 힌트 | 각 선택의 확실한 결과와 불확실한 결과를 분리 표기 |
| 관계 변화 | 없음 |
| 색상 반응 | 사용인 서명은 선택 장치에서 제외되고 코어 외곽에 머묾 |
| 감각 연출 | 선택 직전 배경음이 사라지고 연필 긁는 소리만 남음 |
| 구현 메모 | EDC는 F3 조사 결과를 재판정하지 않는다. 시스템 저장 후 final_decision과 final_choice_relation을 같은 트랜잭션에서 확정 |

표시 정보:

| 선택 | 확실함 | 불확실함 |
| --- | --- | --- |
| 기상 절차 실행 | 시뮬레이션 연결 해제, 현실 신체 기상 시도 | 외부 환경, 장기 생존, 연구원 인격의 지속 |
| 안정화 루프 복원 | 저택과 사용인 유지, 주인공의 인지 연속성 보존 | 장기 시스템 수명, 감정 둔화, 외부 회복 기회 |

### 2.1 F3 인계와 중립 포커스

```text
F2 완료
→ F3_ENTRY: final_sleep_lock=true
→ 냉각 기상 장치·루프 안정화 장치·주인공 수첩 조사
→ F3_BALANCED_SUMMARY
→ SAVE_F3_COMPLETE
→ F3_complete=true
→ EDC
```

- F3_ENTRY 이후 세계 내 수면은 비활성화한다.
- EDC 취소는 F3 자유 조사로 돌아가며 `final_sleep_lock`을 해제하지 않는다.
- 저장·불러오기와 F3 정보 재열람은 허용한다.
- EDC 진입 시 기본 포커스는 두 선택지 사이의 중립 SUBJECT 표식이다.
- 두 선택지의 버튼 크기, 밝기, 거리, 위험 아이콘 수, 확인 횟수를 동일하게 한다.
- F3에서 마지막으로 본 장치와 F0-E 임시 의향을 포커스·카메라·추천 문구에 사용하지 않는다.

### 2.2 F0-E 의향 처리

`f0_provisional_intent`:

| 값 | 의미 | EDC 영향 |
| --- | --- | --- |
| reality | 당시 현실 쪽으로 마음이 기울었음 | 없음 |
| stay | 당시 잔류 쪽으로 마음이 기울었음 | 없음 |
| undecided | 당시 결정을 보류함 | 없음 |

EDC 공정성 규칙:

- 두 선택지를 항상 동시에 활성화한다.
- 이전 의향과 같은 선택지에 기본 포커스를 두지 않는다.
- 버튼 크기·색·밝기·거리·진동을 동일하게 한다.
- 이전 의향을 선택 버튼이나 경고 문구에 표시하지 않는다.
- 사용인은 선택 전에 의향을 알거나 반응하지 않는다.
- 이전 의향과 다른 선택에도 추가 확인·사과·관계 감소를 요구하지 않는다.

### 2.3 최종 선택 관계

| F0-E 의향 | 현실 선택 | 잔류 선택 |
| --- | --- | --- |
| reality | reaffirmed | revised |
| stay | revised | reaffirmed |
| undecided | formed | formed |

`final_choice_relation`은 주인공의 확인 독백만 바꾼다.

| 값 | 독백 |
| --- | --- |
| reaffirmed | “그때 기울었던 쪽을 지금 선택한다.” |
| revised | “생각은 달라졌다. 지금의 선택은 이것이다.” |
| formed | “이제 정한다.” |

세 문장은 가치 판단 없이 같은 길이·음량·연필 연출을 사용한다.

### 2.4 상태 커밋

```yaml
event_id: EDC
guards:
  all:
    - F3_complete
    - final_sleep_lock == true
    - subject_authority_restored
    - final_decision == unset
inputs:
  provisional_intent: f0_provisional_intent
  selected_decision: reality_or_stay
outputs:
  set:
    final_decision: selected_decision
    final_choice_relation: derived_relation
```

파생 규칙:

```text
provisional_intent == selected_decision → reaffirmed
provisional_intent == undecided        → formed
그 외                                  → revised
```

### 2.5 여섯 경로 QA

1. reality→reality: reaffirmed, ED_REALITY.
2. reality→stay: revised, ED_STAY.
3. stay→reality: revised, ED_REALITY.
4. stay→stay: reaffirmed, ED_STAY.
5. undecided→reality: formed, ED_REALITY.
6. undecided→stay: formed, ED_STAY.

모든 경로에서:

- 두 선택지가 모두 활성화된다.
- 관계·기록·결산 단계가 바뀌지 않는다.
- 엔딩 공통 내용과 사용인 반응은 최종 선택만 읽는다.
- `final_choice_relation`은 엔딩 도입의 한 문장 외에는 사용하지 않는다.
- EDC 취소 뒤 F3로 돌아가도 `final_decision=unset`, `final_choice_relation=unresolved`, `final_sleep_lock=true`를 유지한다.

확인 문장:

```text
나는 결과를 보장받지 못했다.
그래도 이 선택이 내 것임을 확인한다.
```

## 3. ED_REALITY 기상 절차

### 공통 이벤트

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 코어실에서 현실 냉각실로 전환 |
| 선행 조건 | `final_decision == reality` |
| 목표 | 시뮬레이션을 떠나 황폐한 현실에 홀로 섬 |
| 상호작용 | 마지막으로 뒤를 보거나 즉시 기상 진행 |
| 성공·실패 | 기상은 성공하지만 생존 결과는 열린 결말 |
| 힌트 | 없음 |
| 관계 변화 | 없음 |
| 색상 반응 | 서명은 연결 종료 순서대로 분리되고 화면에는 흑연 표시만 남음 |
| 감각 연출 | 처음 맡는 실제 먼지 냄새, 통증이 있는 호흡, 무게를 가진 햇빛 |
| 구현 메모 | 6~12분; 외부 생존 여부를 확정하지 않음 |

진행:

1. 주인공이 수첩에 기상 명령을 쓴다.
2. 연구원 인격의 코어 연결을 유지할지 묻는 시스템 창이 나타난다.
3. 주인공은 연결을 즉시 파괴하지 않고 저전력 보존 상태로 둔다.
4. 사용인들은 관계 단계에 따라 작별한다.
5. 화면이 꺼진 뒤 냉각 캡슐 내부에서 눈을 뜬다.
6. 문을 열면 폐허가 된 지표와 멀리 불명확한 신호등이 보인다.
7. 주인공은 수첩의 마지막 장을 찢어 바깥 표지로 사용하고 걸음을 내딛는다.

연구원 인격을 즉시 소멸시키지 않는 이유는 선택의 윤리적 부담을 주인공에게 강제하지 않기 위해서다. 현실에서 다시 접속할 수 있는지는 확정하지 않는다.

### 관계 단계 변형

| 단계 | 장면 |
| --- | --- |
| LOW | 에드가가 공식 종료문을 읽고, 다른 서명은 멀리서 짧게 점멸 |
| MID | 완료한 사용인이 직접 작별하고 미완료 사용인의 기록음이 배경에 남음 |
| HIGH | 네 명이 두려움과 축복을 각자의 말로 표현 |
| ALL | 다섯 명이 서로의 이름을 확인한 뒤 주인공의 선택을 공동 승인 |

### 사용인별 반응

| 사용인 | 핵심 완료 시 | 미완료 시 |
| --- | --- | --- |
| 에드가 | “이번 명령은 제가 아니라 당신이 내렸습니다.” | “기상 절차를 집행합니다, 아가씨.” |
| 마라 1 | 주황 흔적을 수첩 가장자리에 남김 | 마지막 먼지를 닦고 물러남 |
| 루카 | 현실 호흡법과 캡슐 해제 순서를 알려 줌 | 생체 수치만 읽어 줌 |
| 이리스 | 외부의 실제 계절을 대신 확인해 달라고 부탁 | “모델과 다를 수 있습니다”라고 경고 |
| 마라 2 | 자신의 이름을 잊지 말라고 직접 부탁 | 보라 익명 인덱스가 빠른 3음만 남김 |

### IRIS_ED_REALITY

`IRIS_ED_REALITY`는 엔딩 진입 시 `iris_confession_state`를 다시 계산해 아래 작별을 고른다.

| 상태 | 현실 기상 반응 |
| --- | --- |
| `inferred_only` | 외부 계절 모델이 실제 관측이 아님을 경고하고 판단을 주인공에게 돌림 |
| `denied` | 로그 해석을 끝까지 부정하지만 캡슐 해제 절차를 방해하지 않음 |
| `withheld` | 감금 책임과 위험한 희망을 인정하고, 말하지 못한 부분을 남긴 채 작별 |
| `indirect` | 주인공이 사라지면 끝날 것이라 생각했던 자신을 우회적으로 다시 언급 |
| `direct_private` | 사적 고백을 반복 공개하지 않고 “그 말을 잊지 않아도 됩니다”라고 확인 |
| `public` | 모두 앞에서 자신의 바람과 책임을 다시 인정하고 실제 계절을 확인해 달라고 부탁 |

이 반응은 `final_decision`이나 엔딩 단계로 고백 상태를 바꾸지 않는다.

### MARA2_ED_REALITY

- `E3_5_complete=false`: 보라 이중 윤곽은 이름 없이 남지만, 주인공이 P3B의 이름표 배치를 수첩 여백에 다시 그린다.
- `servants.MARA2.archive_resolution=merged`: 장난스러운 목소리로 “밖이 재미없으면 후기를 남겨!”라고 말한 뒤 마지막 음이 떨린다.
- `servants.MARA2.archive_resolution=separated`: 원본과 주석이 동시에 “둘 다 기억해”라고 말한다.
- `mara2_name_written=true`: 위 분기 뒤 현실의 수첩 첫 장에 임시 호칭 클로즈업을 추가한다.

`E3_5_complete=true`인데 `archive_resolution=none`이면 미완료 대사로 대체하지 않고 저장 무결성 오류를 보고한다.

### 마지막 화면

색보정 없는 회갈색 현실 위에서 수첩의 흑연선만 선명하다. 멀리 보이는 점멸 신호는 사람, 자동 장치, 오류 중 무엇인지 확인하지 않는다.

엔딩 표기:

```text
ED_A 현실 기상
FINAL DECISION: REALITY
```

## 4. ED_STAY 안정화 루프 복원

### 공통 이벤트

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 코어실에서 복원된 저택의 저녁으로 전환 |
| 선행 조건 | `final_decision == stay` |
| 목표 | 거짓임을 인지한 상태로 시뮬레이션에 남음 |
| 상호작용 | 유지할 기억 범위와 저택 외형 선택 |
| 성공·실패 | 루프 안정화 성공. 장기 결과는 열린 결말 |
| 힌트 | 감각 둔화 위험과 시스템 자원 소모를 명시 |
| 관계 변화 | 없음 |
| 색상 반응 | 다섯 서명은 장식 속에 숨지 않고 고딕 외형과 공존 |
| 감각 연출 | 익숙한 음식 향이 돌아오지만 금속 뒷맛은 완전히 사라지지 않음 |
| 구현 메모 | 7~13분; 기억 삭제를 강요하지 않음 |

잔류 시 세부 선택:

| 항목 | 권장 기본 | 효과 |
| --- | --- | --- |
| 진실 기억 | 유지 | 주인공은 세계가 시뮬레이션임을 계속 인지 |
| 파열 흔적 | 일부 유지 | 색 서명과 SF 골격이 드문 순간 노출 |
| 루프 길이 | 유연한 하루 | 고정 시각표가 아닌 사건 완료 기반 수면 |
| 사용인 자율성 | 확대 | 각자 주 공간과 대화 시간을 스스로 선택 |

주인공이 진실 기억을 지우는 선택지는 제공하지 않는다. 잔류의 핵심은 기만당한 상태로 돌아가는 것이 아니라, 불완전한 가상 삶을 알고도 선택하는 데 있다.

### 관계 단계 변형

| 단계 | 장면 |
| --- | --- |
| LOW | 주방 식탁 양옆에 사용인이 서고 대화가 업무 문장으로 제한 |
| MID | 완료한 사용인만 자리에 앉으며 미완료 사용인은 역할을 수행 |
| HIGH | 네 명이 주인공과 같은 식탁에 앉고 루프 규칙 변경에 참여 |
| ALL | 다섯 명 전원이 각자 색과 이름을 유지한 채 새 하루의 규칙을 공동 작성 |

### 사용인별 반응

| 사용인 | 핵심 완료 시 | 미완료 시 |
| --- | --- | --- |
| 에드가 | 수면 권유를 명령이 아닌 질문으로 바꿈 | 기존 취침 방송을 낮은 음성으로 반복 |
| 마라 1 | 일부 얼룩을 지우지 않고 날짜로 남김 | 저택을 완벽히 복원하려 함 |
| 루카 | 차의 온도를 매일 다르게 설정 | 동일 레시피를 유지 |
| 이리스 | 계절 장치를 실제 외부 센서와 느슨하게 연결 | 이상적인 봄을 반복 |
| 마라 2 | 초상화 이름표를 매일 주인공과 바꾸어 봄 | 자신의 이름표를 반복 확인 |

### IRIS_ED_STAY

`IRIS_ED_STAY`도 `IRIS_ED_REALITY`와 같은 `iris_confession_state` 계산 결과를 읽되 잔류 이후의 책임 방식만 다르게 보여 준다.

| 상태 | 안정화 잔류 반응 |
| --- | --- |
| `inferred_only` | 이상적인 봄을 유지하되 외부 센서값과 연출값을 주인공이 직접 비교할 수 있게 함 |
| `denied` | 로그 해석은 부정하면서도 계절 장치 변경 권한을 주인공에게 반환 |
| `withheld` | 감금 책임을 인정하고 계절 규칙의 모든 변경 이력을 공개 |
| `indirect` | 주인공이 없는 고요를 바라지 않도록 계절 장치 중단권을 공동 규칙에 넣음 |
| `direct_private` | 사적 고백을 공동 규칙으로 이용하지 않고, 숨기는 순간 지적해 달라고 요청 |
| `public` | 다섯 인격 앞에서 책임을 재확인하고 실제 외부 센서와 계절 규칙을 공동 작성 |

잔류 선택은 고백의 보상이나 용서가 아니며 판정값을 별도 저장하지 않는다.

### MARA2_ED_STAY

- `E3_5_complete=false`: 마라 2는 계속 건방지게 굴지만 마지막 3음 중 하나가 비어 있다.
- `servants.MARA2.archive_resolution=merged`: “영원히 기억할 수 있겠네!”라고 과장한 뒤, 영원이라는 말이 무섭다고 작게 덧붙인다.
- `servants.MARA2.archive_resolution=separated`: 두 기록을 서로 다른 액자에 두고 매일 위치를 바꾸는 놀이를 제안한다.
- `mara2_name_written=true`: 위 분기 뒤 주인공 수첩과 아카이브의 이름 필드 이중 저장 장면을 추가한다.

현실·잔류 엔딩은 같은 `archive_resolution`을 읽으며 엔딩 선택이나 관계 수치로 보존 방식을 다시 계산하지 않는다.

### 마지막 장면

글리치가 일부 남은 주방. 주인공은 식탁 정면에 앉는다. 카메라는 과거 잔류안처럼 사용인이 양옆에 선 구도를 먼저 보여 준 뒤, 관계 단계에 따라 앉는 사람 수를 바꾼다.

주인공이 수첩에 쓴다.

```text
내일도 같은 날일 수 있다.
하지만 같은 선택일 필요는 없다.
```

엔딩 표기:

```text
ED_B 안정화 잔류
FINAL DECISION: STAY
```

## 5. ALL·전원 완료 추가 장면

`all_servants_complete`일 때만 두 엔딩에 공통으로 삽입한다.

1. 다섯 연구원 인격이 고유 색·문양·음향으로 자신의 이름을 인증한다.
2. 마라 2가 체크섬을 읽어 모두의 기록이 서로에게 분산 백업되었음을 확인한다.
3. 에드가가 주인공의 SUBJECT 권한이 어떤 사용인 권한보다 높다고 선언한다.
4. 주인공은 이 선언을 수첩에 자기 문장으로 다시 써야 최종 실행할 수 있다.

이 장면은 연구원 해방 성공 엔딩이 아니다. 다섯 인격이 서로를 독립된 사람으로 인정하고 주인공의 선택을 침해하지 않는 상태를 결산한다.

## 6. 엔딩 후 처리

- 엔딩 크레딧 뒤 `F3 마지막 확인` 시스템 저장을 불러올 수 있다.
- 재선택은 세계관 내부 되감기가 아니라 사용자 편의 기능으로 표시한다.
- 완료 갤러리에는 선택하지 않은 엔딩의 이미지나 대사를 미리 공개하지 않는다.
- 관계 이벤트 회수용 챕터 선택은 1회 엔딩 후 해금하며, 본편 상태와 별도 슬롯을 사용한다.
