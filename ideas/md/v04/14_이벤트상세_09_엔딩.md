# GGB v0.4 이벤트 상세 09: 엔딩

## 1. 설계 원칙

- 현실은 용기, 잔류는 안정이라는 단순 도덕 판정을 하지 않는다.
- 현실은 불확실하고 위험하며, 잔류는 거짓임을 알고 선택하는 삶이다.
- 관계 이벤트는 선택지의 존재를 막지 않고 작별의 해상도만 바꾼다.
- 주인공의 SUBJECT 권한은 연필·종이·필기음으로 표현한다.
- 사용인의 색은 최종 반응에서 분리·공존·소실 상태를 보여 주지만 선택 UI의 정답 표시는 하지 않는다.

## 2. EDC 최종 선택

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 코어실, F3 이후 |
| 선행 조건 | `subject_authority_restored`, `final_decision == unset` |
| 목표 | 현실 기상 또는 안정화 루프 복원 선택 |
| 상호작용 | 두 장치 조사, 선택 문장 자필 확인, 최종 실행 |
| 성공·실패 | 취소하여 F3 조사로 돌아갈 수 있음. 실행 후 확정 |
| 힌트 | 각 선택의 확실한 결과와 불확실한 결과를 분리 표기 |
| 관계 변화 | 없음 |
| 색상 반응 | 사용인 서명은 선택 장치에서 제외되고 코어 외곽에 머묾 |
| 감각 연출 | 선택 직전 배경음이 사라지고 연필 긁는 소리만 남음 |
| 구현 메모 | 시스템 저장 후 분기. `final_decision = reality|stay` |

표시 정보:

| 선택 | 확실함 | 불확실함 |
| --- | --- | --- |
| 기상 절차 실행 | 시뮬레이션 연결 해제, 현실 신체 기상 시도 | 외부 환경, 장기 생존, 연구원 인격의 지속 |
| 안정화 루프 복원 | 저택과 사용인 유지, 주인공의 인지 연속성 보존 | 장기 시스템 수명, 감정 둔화, 외부 회복 기회 |

확인 문장:

```text
나는 결과를 보장받지 못했다.
그래도 이 선택이 내 것임을 확인한다.
```

## 3. ED_REALITY 기상 절차

### 공통 이벤트

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 코어실에서 현실 냉각실로 전환 |
| 선행 조건 | `final_decision == reality` |
| 목표 | 시뮬레이션을 떠나 황폐한 현실에 홀로 섬 |
| 상호작용 | 마지막으로 뒤를 보거나 즉시 기상 진행 |
| 성공·실패 | 기상은 성공하지만 생존 결과는 열린 결말 |
| 힌트 | 없음 |
| 관계 변화 | 없음 |
| 색상 반응 | 서명은 연결 종료 순서대로 분리되고 화면에는 흑연 표시만 남음 |
| 감각 연출 | 처음 맡는 실제 먼지 냄새, 통증이 있는 호흡, 무게를 가진 햇빛 |
| 구현 메모 | 6~12분; 외부 생존 여부를 확정하지 않음 |

진행:

1. 주인공이 수첩에 기상 명령을 쓴다.
2. 연구원 인격의 코어 연결을 유지할지 묻는 시스템 창이 나타난다.
3. 주인공은 연결을 즉시 파괴하지 않고 저전력 보존 상태로 둔다.
4. 사용인들은 관계 단계에 따라 작별한다.
5. 화면이 꺼진 뒤 냉각 캡슐 내부에서 눈을 뜬다.
6. 문을 열면 폐허가 된 지표와 멀리 불명확한 신호등이 보인다.
7. 주인공은 수첩의 마지막 장을 찢어 바깥 표지로 사용하고 걸음을 내딛는다.

연구원 인격을 즉시 소멸시키지 않는 이유는 선택의 윤리적 부담을 주인공에게 강제하지 않기 위해서다. 현실에서 다시 접속할 수 있는지는 확정하지 않는다.

### 관계 단계 변형

| 단계 | 장면 |
| --- | --- |
| LOW | 에드가가 공식 종료문을 읽고, 다른 서명은 멀리서 짧게 점멸 |
| MID | 완료한 사용인이 직접 작별하고 미완료 사용인의 기록음이 배경에 남음 |
| HIGH | 네 명 이상이 두려움과 축복을 각자의 말로 표현 |
| 전원 완료 | 다섯 명이 서로의 이름을 확인한 뒤 주인공의 선택을 공동 승인 |

### 사용인별 반응

| 사용인 | 핵심 완료 시 | 미완료 시 |
| --- | --- | --- |
| 에드가 | “이번 명령은 제가 아니라 당신이 내렸습니다.” | “기상 절차를 집행합니다, 아가씨.” |
| 마라 1 | 주황 흔적을 수첩 가장자리에 남김 | 마지막 먼지를 닦고 물러남 |
| 루카 | 현실 호흡법과 캡슐 해제 순서를 알려 줌 | 생체 수치만 읽어 줌 |
| 이리스 | 외부의 실제 계절을 대신 확인해 달라고 부탁 | “모델과 다를 수 있습니다”라고 경고 |
| 마라 2 | 자신의 이름을 잊지 말라고 직접 부탁 | 보라 익명 인덱스가 빠른 3음만 남김 |

### MARA2_ED_REALITY

- `mara2_name_written`: 현실의 수첩 첫 장에 임시 호칭이 남아 있다.
- 병합: 장난스러운 목소리로 “밖이 재미없으면 후기를 남겨!”라고 말한 뒤 마지막 음이 떨린다.
- 분리 보존: 원본과 주석이 동시에 “둘 다 기억해”라고 말한다.
- 미완료: 보라 이중 윤곽은 이름 없이 남지만, 주인공이 P3B의 이름표 배치를 수첩 여백에 다시 그린다.

### 마지막 화면

색보정 없는 회갈색 현실 위에서 수첩의 흑연선만 선명하다. 멀리 보이는 점멸 신호는 사람, 자동 장치, 오류 중 무엇인지 확인하지 않는다.

엔딩 표기:

```text
ED_A 현실 기상
FINAL DECISION: REALITY
```

## 4. ED_STAY 안정화 루프 복원

### 공통 이벤트

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 코어실에서 복원된 저택의 저녁으로 전환 |
| 선행 조건 | `final_decision == stay` |
| 목표 | 거짓임을 인지한 상태로 시뮬레이션에 남음 |
| 상호작용 | 유지할 기억 범위와 저택 외형 선택 |
| 성공·실패 | 루프 안정화 성공. 장기 결과는 열린 결말 |
| 힌트 | 감각 둔화 위험과 시스템 자원 소모를 명시 |
| 관계 변화 | 없음 |
| 색상 반응 | 다섯 서명은 장식 속에 숨지 않고 고딕 외형과 공존 |
| 감각 연출 | 익숙한 음식 향이 돌아오지만 금속 뒷맛은 완전히 사라지지 않음 |
| 구현 메모 | 7~13분; 기억 삭제를 강요하지 않음 |

잔류 시 세부 선택:

| 항목 | 권장 기본 | 효과 |
| --- | --- | --- |
| 진실 기억 | 유지 | 주인공은 세계가 시뮬레이션임을 계속 인지 |
| 파열 흔적 | 일부 유지 | 색 서명과 SF 골격이 드문 순간 노출 |
| 루프 길이 | 유연한 하루 | 고정 시각표가 아닌 사건 완료 기반 수면 |
| 사용인 자율성 | 확대 | 각자 주 공간과 대화 시간을 스스로 선택 |

주인공이 진실 기억을 지우는 선택지는 제공하지 않는다. 잔류의 핵심은 기만당한 상태로 돌아가는 것이 아니라, 불완전한 가상 삶을 알고도 선택하는 데 있다.

### 관계 단계 변형

| 단계 | 장면 |
| --- | --- |
| LOW | 주방 식탁 양옆에 사용인이 서고 대화가 업무 문장으로 제한 |
| MID | 완료한 사용인만 자리에 앉으며 미완료 사용인은 역할을 수행 |
| HIGH | 네 명 이상이 주인공과 같은 식탁에 앉고 루프 규칙 변경에 참여 |
| 전원 완료 | 다섯 명 전원이 각자 색과 이름을 유지한 채 새 하루의 규칙을 공동 작성 |

### 사용인별 반응

| 사용인 | 핵심 완료 시 | 미완료 시 |
| --- | --- | --- |
| 에드가 | 수면 권유를 명령이 아닌 질문으로 바꿈 | 기존 취침 방송을 낮은 음성으로 반복 |
| 마라 1 | 일부 얼룩을 지우지 않고 날짜로 남김 | 저택을 완벽히 복원하려 함 |
| 루카 | 차의 온도를 매일 다르게 설정 | 동일 레시피를 유지 |
| 이리스 | 계절 장치를 실제 외부 센서와 느슨하게 연결 | 이상적인 봄을 반복 |
| 마라 2 | 초상화 이름표를 매일 주인공과 바꾸어 봄 | 자신의 이름표를 반복 확인 |

### MARA2_ED_STAY

- 병합: “영원히 기억할 수 있겠네!”라고 과장한 뒤, 영원이라는 말이 무섭다고 작게 덧붙인다.
- 분리 보존: 두 기록을 서로 다른 액자에 두고 매일 위치를 바꾸는 놀이를 제안한다.
- `mara2_name_written`: 주인공 수첩과 아카이브에 이름 필드가 이중 저장된다.
- 미완료: 마라 2는 계속 건방지게 굴지만 마지막 3음 중 하나가 비어 있다.

### 마지막 장면

글리치가 일부 남은 주방. 주인공은 식탁 정면에 앉는다. 카메라는 과거 잔류안처럼 사용인이 양옆에 선 구도를 먼저 보여 준 뒤, 관계 단계에 따라 앉는 사람 수를 바꾼다.

주인공이 수첩에 쓴다.

```text
내일도 같은 날일 수 있다.
하지만 같은 선택일 필요는 없다.
```

엔딩 표기:

```text
ED_B 안정화 잔류
FINAL DECISION: STAY
```

## 5. 전원 완료 추가 장면

`all_servants_complete`일 때만 두 엔딩에 공통으로 삽입한다.

1. 다섯 연구원 인격이 고유 색·문양·음향으로 자신의 이름을 인증한다.
2. 마라 2가 체크섬을 읽어 모두의 기록이 서로에게 분산 백업되었음을 확인한다.
3. 에드가가 주인공의 SUBJECT 권한이 어떤 사용인 권한보다 높다고 선언한다.
4. 주인공은 이 선언을 수첩에 자기 문장으로 다시 써야 최종 실행할 수 있다.

이 장면은 연구원 해방 성공 엔딩이 아니다. 다섯 인격이 서로를 독립된 사람으로 인정하고 주인공의 선택을 침해하지 않는 상태를 결산한다.

## 6. 엔딩 후 처리

- 엔딩 크레딧 뒤 `F3 마지막 확인` 시스템 저장을 불러올 수 있다.
- 재선택은 세계관 내부 되감기가 아니라 사용자 편의 기능으로 표시한다.
- 완료 갤러리에는 선택하지 않은 엔딩의 이미지나 대사를 미리 공개하지 않는다.
- 관계 이벤트 회수용 챕터 선택은 1회 엔딩 후 해금하며, 본편 상태와 별도 슬롯을 사용한다.
