# GGB v0.4 기획 문서

## 1. 개요

이 폴더는 GGB(가칭)의 v0.4 기획 문서 세트다.

GGB는 멸망 이후 지구의 냉각 시설과 고딕 저택 시뮬레이션을 오가는 심리 서사형 포인트 앤 클릭 어드벤처다. 플레이어는 같은 하루를 반복하면서 비가역 퍼즐을 수면 리셋으로 다시 시도하고, 이전 루프를 기억하는 다섯 사용인과 관계를 맺는다.

핵심 요소:

- 수면에 의한 물리 상태 리셋.
- 리셋 뒤에도 유지되는 수첩·지식·관계·사용인 기억.
- 반복 작업을 줄이는 영구 숏컷.
- 고딕 저택에서 SF 시설로 이어지는 점진적 파열.
- 인간 연구원의 인격을 가진 다섯 사용인.
- 인격 데이터 출처를 나타내는 색상 서명.
- 황폐한 현실과 자각된 시뮬레이션 사이의 최종 선택.

본 폴더의 문서는 프로토타입이나 실행 코드가 아니라 상세 게임 기획을 정의한다. 추후 엔진은 Godot을 사용할 예정이다.

## 5. 핵심 확정사항

### 리셋

- D5 이전 잠들면 같은 침실의 같은 아침으로 돌아간다.
- 물리 상태·시간·인벤토리는 초기화된다.
- 수첩·일지·지식·관계·사용인 잔류 기억은 유지된다.
- HARD FAILURE는 B3-B, C4, D1에만 사용한다.
- D5 이후 정상 리셋은 복구되지 않는다.

### 관계

- `bond`와 `alert`는 각각 `0~5`이며 독립값이다.
- 관계 이벤트는 F0나 엔딩 선택지를 잠그지 않는다.
- 연구원 기록 수와 관계 완료 수는 별도 값이다.
- 사용인은 이전 루프를 기억하지만 자기 담당 영역과 직접 목격을 중심으로 기억한다.

### 사용인

| 사용인 | 아바타 | 대표 기능 |
| --- | --- | --- |
| 에드가 | 용 | 운영·보안·코어 |
| 마라 1 | 여우 | 정비·배선·표층 유지 |
| 루카 | 쥐 | 생명 유지·냉각 |
| 이리스 | 인공 천사 | 외부 환경·계절 |
| 마라 2 | 박쥐·흡혈마 | 인격 아카이브·체크섬 |

원래 연구원은 모두 인간이다. 종족 특징은 시뮬레이션 아바타와 기능 인터페이스다.

### 색상 서명

- 색은 인격 데이터의 출처를 나타낸다.
- 색만으로 필수 퍼즐을 풀게 하지 않는다.
- 문양, 선 패턴, 음향, 텍스트 라벨을 병행한다.
- 주인공의 SUBJECT 권한은 흑연·종이·필기음으로 표현한다.

### 엔딩

- 현실 기상과 안정화 잔류 모두 선택 가능하다.
- 관계 완료 수는 엔딩 반응을 바꾸지만 선택지를 잠그지 않는다.
- 현실은 안전을 보장하지 않는다.
- 잔류는 진실 기억을 삭제하지 않는다.

## 6. 주요 용어

| 용어 | 의미 |
| --- | --- |
| `NORMAL_RESET` | D5 이전 정상 물리 초기화 |
| `BROKEN_RESET` | D5 이후 고딕 필터 복구 실패 |
| `LOCAL RETRY` | 수면 없이 현장에서 재시도 |
| `HARD FAILURE` | 당일 물리 상태가 비가역적으로 변해 수면 필요 |
| 영구 정보 | 정상 리셋 뒤에도 유지되는 지식·기록 |
| 검증 부분 | 퍼즐에서 정답임이 확인된 하위 단계 |
| 숏컷 | 검증된 준비·이동·일과의 축약 |
| 잔류 기억 | 사용인이 리셋 뒤에도 보유하는 사건·감정 패턴 |
| 개입 예산 | 사용인이 한 루프에서 사용할 수 있는 질문·제지 횟수 |
| 색상 서명 | 인격 데이터의 소유자 식별 묶음 |
| SUBJECT | 최종 결과를 겪고 선택하는 주인공 권한 |

## 7. ID 읽기

| 형태 | 의미 |
| --- | --- |
| `P1`, `C4`, `E3_5` | 실제 콘텐츠 이벤트 |
| `J1~J5` | 일지 복원 |
| `ED_*` | 엔딩·엔딩 반응 |
| `CLR-*` | 색상 서명 학습 단계 |
| `MARA2_*` | 마라 2 전용 하위 이벤트 |
| `CHK_*` | 흐름도의 조건 판정 |
| `REL_*` | 관계 반응 결과 |
| `SYS_*` | 저장·집계·동기화 처리 |

`CHK_*`, `REL_*`, `SYS_*`는 모두 별도 플레이 콘텐츠를 뜻하지 않는다. 실제 등록 여부는 [04 이벤트·상태표](04_전체이벤트리스트_상태표.md)를 따른다.

## 8. 문서 작성 원칙

- 최신 확정사항을 이전 문서보다 우선한다.
- 한 문서는 자신이 소유한 영역만 확정한다.
- 선택 이벤트를 메인 필수 게이트로 만들지 않는다.
- 퍼즐 정답 변경은 `24-3`과 `08`을 함께 검토한다.
- 새 이벤트 ID는 `04`와 `17`에 등록한다.
- 색상 단서는 반드시 비색상 대체를 가진다.
- 관계값 변화는 같은 이벤트에서 최초 1회만 적용한다.
- 미확정 설정은 추론해 확정하지 않는다.

## 9. 검증 원칙

문서 수정 후 확인:

1. 상대 링크가 존재하는가.
2. 코드 펜스와 Mermaid 블록이 닫혔는가.
3. 이벤트 ID와 플래그 표기가 일치하는가.
4. 관계 이벤트 미완료 경로가 유지되는가.
5. 실패 뒤 재진입점이 존재하는가.
6. D5 이후 NORMAL_RESET으로 돌아가지 않는가.
7. 색 제거·음량 0에서도 필수 진행이 가능한가.
