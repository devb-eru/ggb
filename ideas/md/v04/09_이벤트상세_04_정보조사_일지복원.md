# GGB v0.4 이벤트 상세 04: 정보 조사·일지 복원

## 1. 문서 목적

본 문서는 플레이어가 정보를 발견하고, 수첩에 가설을 세우고, 퍼즐과 시스템 기록으로 검증하는 과정을 정의한다.

담당 범위:

- P3에서 아버지 일지의 존재를 발견하는 정보 인계.
- B1 시간표 조사와 B2 기록 내실 진입.
- J1~J5 일지 복원 사건.
- B4 파형, C2·C2-1 기록, C5 진단 자료의 획득과 해석.
- D0 지하 단서 재확인.
- J4 기록량 변형과 F1 아버지 마지막 기록.
- 정보의 `unknown`부터 `authenticated`까지 수명 주기.
- 수첩, 출처, 재열람, 접근성, 저장과 QA 규칙.

담당하지 않는 범위:

- B3, C3, C4, D0-A, D1, D4, F0의 정답과 조작 규칙.
- 사용인 핵심 관계 이벤트의 감정 선택과 관계값.
- E5, F2, F3, EDC와 엔딩 연출.
- 실제 Godot 리소스 구현.

퍼즐 정답은 `08`, 공간 ID는 `05`, 루프 저장은 `07`, 관계 장면은 `11~14`의 책임을 유지한다.

### 2.2 J4 진행 순서

```text
E_HUB 조사 종료
→ J4
→ E3_4_complete 검사
→ 미완료면 E3_4M
→ E5
```

J4는 E3_4나 E3_4M을 선행 조건으로 요구하지 않는다.

### 2.3 관계 독립

- 연구원 기록 0개여도 J4 BASE를 읽을 수 있다.
- 관계 완료 수와 연구원 기록 수는 별도 값이다.
- 관계 상태는 정보의 양과 어조를 바꿀 수 있지만 필수 사실을 삭제하지 않는다.
- 마라 2 기록이 없어도 익명 보라 인덱스를 통해 F0-D에 진입한다.
- F1과 J5는 관계 수와 무관하게 필수로 발생한다.

### 2.4 실패 대책

- 정보 조사는 플레이어를 막는 강제 실패를 사용하지 않는다.
- 잘못된 추론은 `LOCAL RETRY` 또는 가설 상태로 남긴다.
- B2에서 에드가에게 발각되어도 퇴장, 당일 잠금, 리셋을 발생시키지 않는다.
- 일지 복원 오배열은 페이지가 원위치로 돌아가며 즉시 재시도한다.
- 이미 획득한 필수 정보는 사용인 행동으로 삭제되지 않는다.

## 3. 정보 매체

| 매체 | 주체 | 기능 | 신뢰 한계 |
| --- | --- | --- | --- |
| 주인공 수첩 | SUBJECT | 관찰, 가설, 자기 작성, 퍼즐 작업 | 관찰과 해석을 구분해야 함 |
| 아버지 일지 | CREATOR | 백도어 단서, 책임 기록, 마지막 고백 | J1~J4는 손상·편집 가능성 존재 |
| 연구원 기록 | RESIDENT | 사용인의 동의 범위, 기억, 원망 | 개인 관점과 누락이 존재 |
| 색상 서명 | RESIDENT 내부 출처 | 누가 남긴 기록인지 확인 | 역할 정답을 직접 제공하지 않음 |
| 시스템 진단 | SYSTEM | 신체·환경·권한 상태 | 의미 해석에는 다른 기록 필요 |
| 에드가 접근 기록 | CUSTODIAN | 문, 감시, 운영 권한 | 보호 목적의 은폐가 섞임 |
| 현재 자기 작성 | SUBJECT | 현재 선택 주체 인증 | 과거 기록을 대신하지 않음 |

## 4. 정보 수명 주기

### 4.1 상태

| 단계 | 내부 상태 | 수첩 표시 | 사용 가능 범위 |
| --- | --- | --- | --- |
| 미발견 | `unknown` | 없음 | 조건 판정 불가 |
| 관찰 | `observed` | 사실 문장 | 대화·조사 조건 |
| 가설 | `hypothesized` | 물음표와 연결선 | 시험 입력 가능 |
| 검증 | `verified` | 확인 표시 | 숏컷·퍼즐 하위 단계 |
| 인증 | `authenticated` | 출처 서명과 자물쇠 해제 | 권한·F0 역할 판정 |

### 4.2 상태 변화 원칙

```mermaid
flowchart LR
    U["unknown"] --> O["observed"]
    O --> H["hypothesized"]
    H --> V["verified"]
    V --> A["authenticated"]
    O --> V
    H --> O
```

- 관찰 없이 가설을 저장하지 않는다.
- 오답 가설은 삭제하지 않고 `반증됨` 주석을 붙여 보관함으로 이동한다.
- 퍼즐 성공이 모든 관련 사실을 인증하지는 않는다.
- J1~J3의 문장은 영구 저장되지만 아버지 원본임이 F1 전까지 완전히 인증되지는 않는다.
- C5 시스템 진단은 출처가 인증됐지만 의미 해석은 관찰 또는 가설일 수 있다.
- F1은 아버지 기록의 서명과 수정 이력을 인증한다.
- J5는 주인공의 현재 선택 권한을 인증한다.

### 4.3 영구 커밋

다음 중 하나를 만족하면 수면 뒤에도 유지한다.

- 수첩에 주인공이 직접 적었다.
- 일지 복원 단계가 완료됐다.
- 가역 시험 또는 퍼즐 결과로 검증됐다.
- HARD FAILURE가 원인을 구분했다.
- 시스템이 기록 출처와 권한을 인증했다.
- 관계 이벤트가 연구원 기록을 완성했다.

다음은 유지하지 않는다.

- 화면을 보기만 하고 조사하지 않은 정보.
- 선택하지 않은 대화문.
- 미완료 페이지 배열.
- 제조 중간 물질.
- 시험하지 않은 퍼즐 배치.
- B2의 문, 커튼, 서랍 같은 물리 상태.

## 5. 공통 조사 규칙

### 5.1 상호작용 문법

정보 오브젝트에는 최대 네 행동을 사용한다.

| 행동 | 기능 |
| --- | --- |
| 살핀다 | 표면 정보와 감각 단서 |
| 펼친다·뒤집는다 | 숨은 면과 배열 확인 |
| 겹친다 | 두 자료의 관계 검증 |
| 기록한다 | 수첩에 영구 정보로 커밋 |

핫스폿이 많아도 현재 목표와 관련된 행동은 한 화면에 최대 5개만 활성화한다.

### 5.2 피드백

| 상태 | 피드백 |
| --- | --- |
| observed | 사실만 적고 결론을 쓰지 않음 |
| hypothesized | 연결선 끝에 물음표 |
| verified | 검증한 사건과 출처 병기 |
| authenticated | 시스템 라벨·문양·텍스트를 함께 표시 |
| 반증됨 | 지우지 않고 옅은 취소선과 반증 원인 표시 |

### 5.3 반복 조사

- 첫 조사는 전체 연출과 원문을 제공한다.
- 반복 조사는 핵심 문장과 새 조건만 제공한다.
- 이미 읽은 긴 기록은 `전체 재생`, `핵심 요약`, `관련 단서`로 나눈다.
- 숏컷이 있어도 플레이어가 원하면 원문과 원래 조사를 다시 수행할 수 있다.

## 7. P3_INFO_HANDOFF: 일지 존재 발견

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | `M1_LIBRARY_OUTER`, 프롤로그 아침 |
| 선행 조건 | P1 |
| 목표 | P3 책 정리 중 기록 내실과 아버지 일지의 존재 확인 |
| 플레이 시간 | P3 내부 30~60초 |
| 성공·실패 | 발견만 수행, 실패 없음 |
| 다음 | P3 완료, 이후 B1 동기 |

P3의 책 정리 중 마지막 책이 반납 슬롯에 걸린다. 책을 빼면 유리문 안쪽 책상에 낡은 일지가 보인다.

관찰:

- 표지에는 주인공이 어릴 때 그리던 저택 윤곽이 눌려 있다.
- 책등의 제목은 비어 있지만 아버지 사진의 필기와 같은 획이 있다.
- 기록 내실 유리문에는 열쇠 구멍보다 얇은 수직 잠금선이 있다.
- 에드가는 일지를 보았는지 묻지 않고 책 정리 완료만 확인한다.

수첩:

```text
[관찰] 잠긴 유리문 안쪽에 아버지의 필체와 닮은 일지가 있다.
[관찰] 외부 서고는 열려 있지만 안쪽 기록실은 감시와 잠금이 겹쳐 있다.
```

P3에서는 일지를 열 수 없다. 북쪽 기록 회랑에서도 연결문 핫스폿이 나타나지 않는다.

### 9.2 진입

외부 서고의 유리문 잠금선은 시간표에서 확인한 사건 구간에 한 줄만 꺼져 있다.

```text
[문을 연다]
→ 잠금선이 금속 걸쇠보다 반 박자 늦게 사라짐
→ 기록 내실 진입
→ 문은 닫히지만 잠기지 않음
```

잘못된 시간 구간을 선택한 경우 실제 이동하지 않고 수첩이 B1의 검증된 접근 창을 다시 보여 준다.

### 9.3 공간

활성 오브젝트:

| object_id | 오브젝트 | 기능 |
| --- | --- | --- |
| `OBJ_LIBRARY_JOURNAL_DESK` | 일지 책상 | J1 진입 |
| `OBJ_LIBRARY_INDEX_DRAWERS` | 황동 색인 서랍 | 부가 기록·주의도 상승 |
| `OBJ_LIBRARY_RECORD_CLOCK` | 기록 시계 | 에드가 귀환 예고 |
| `OBJ_LIBRARY_SERVICE_ALCOVE` | 커튼 뒤 점검 벽감 | 숨기 |
| `OBJ_LIBRARY_PORTRAIT_LATCH` | 초상화 패널 | J1 후 연결문 해금 |
| `OBJ_LIBRARY_MAP_FLOOR` | 책상 이중 바닥 | J3 전 잠김, D0에서 사용 |

### 9.4 점검 벽감

`OBJ_LIBRARY_SERVICE_ALCOVE`는 바닥부터 천장까지 이어진 벨벳 커튼 뒤의 얕은 공간이다.

고딕 필터:

- 오래된 지도 두루마리와 이동 사다리 보관소.
- 먼지가 적고 바닥에 평행한 긁힘이 있다.

D5 이후 실제 기능:

- 기록 서버 점검자가 서는 서비스 벽감.
- 커튼은 케이블 차폐막의 고딕 위장이다.

첫 조사 뒤 `b2_hide_discovered=true`가 된다. 발견하지 않아도 발각 대화 경로로 진행할 수 있다.

### 9.5 로컬 주의도

```text
b2_attention_level: 0..3
```

| 행동 | 변화 |
| --- | ---: |
| 일지 표지·페이지 가장자리 조사 | 0 |
| 기록 시계 확인 | 0 |
| 황동 색인 서랍 한 번 열기 | +1 |
| 무거운 사다리 이동 | +1 |
| 잠긴 이중 바닥 반복 조작 | +1 |
| 같은 큰 행동 반복 | 최초 1회만 +1 |

`b2_attention_level`은 현재 B2 장면의 압박 상태이며 영구 경계도와 다르다.

에드가의 기존 `alert`는 진입 시점과 대사 어조만 변형한다.

| EDGAR.alert | 진입 판정 |
| --- | --- |
| 0~1 | 로컬 주의도 3에서 진입 |
| 2~3 | 로컬 주의도 2 뒤 다음 큰 행동에서 진입 |
| 4~5 | 로컬 주의도 2에서 진입, 문 앞 정지 연출 추가 |

랜덤 판정은 사용하지 않는다.

### 9.6 진입 예고

| 단계 | 감각 |
| --- | --- |
| 1 | 외부 서고에서 책 한 권이 닫히는 소리 |
| 2 | 문손잡이가 아주 천천히 원위치로 돌아감 |
| 3 | 낮은 시계음과 꼬리가 바닥을 스치는 소리 |

세 번째 예고 뒤 선택:

```text
[계속 조사한다]
[점검 벽감에 숨는다]
[문 쪽을 바라본다]
```

선택 시간 제한은 없다.

### 9.7 B2_HIDE

조건:

```text
b2_hide_discovered == true
```

연출 비트:

1. 에드가가 문을 열고 멈춘다.
2. 일지 책상과 기록 시계를 확인한다.
3. 색인 서랍 또는 움직인 사다리를 원위치로 돌린다.
4. 경계도에 따른 한 문장을 남긴다.
5. 대시계 점검을 위해 나간다.

첫 연출은 12~20초, 반복은 3~5초 요약으로 축약한다. 숨은 상태에서 QTE나 호흡 게이지를 사용하지 않는다.

경계도별 독백:

| alert | 에드가 |
| --- | --- |
| 0~1 | “기록 정리가 어긋났습니다. 마라 2에게 확인해야겠습니다.” |
| 2~3 | “누군가 찾는 것이 있는 듯합니다. 필요한 기록이라면 요청하는 편이 빠릅니다.” |
| 4~5 | “숨긴 흔적보다, 숨기려는 이유가 더 오래 남습니다. 아가씨께서도 알고 계실 겁니다.” |

에드가는 벽감을 열지 않고 나간다. 높은 경계에서도 플레이어를 발견하거나 B2를 잠그지 않는다.

### 9.8 B2_CAUGHT

플레이어가 숨지 않으면 에드가는 들어와 일정 거리를 유지한 채 묻는다.

기본:

```text
에드가: “찾으시는 책이 있습니까?”
```

대화 선택:

| 주인공 | 에드가 반응 |
| --- | --- |
| “아버지의 일지를 보고 있었어.” | “가족 기록을 읽는 일을 제가 금지할 권한은 없습니다. 다만 기록은 읽는 사람보다 오래 남습니다.” |
| “잠이 오지 않아서.” | “그렇습니까. 그렇다면 더 밝은 열람실을 준비하겠습니다. 이곳은 오래 머물 장소가 아닙니다.” |
| “왜 이 방을 잠가 뒀어?” | “보호해야 할 기록이 있기 때문입니다. 무엇으로부터 보호하는지는, 지금 답해도 믿지 않으실 겁니다.” |
| 침묵 | “대답을 강요하지 않겠습니다. 대신 다치실 만한 서랍은 열지 마십시오.” |

높은 경계 추가 문장:

```text
에드가: “우연은 같은 문 앞에서 반복되지 않습니다.
아가씨께서 무엇을 확인하려는지는 묻지 않겠습니다.
다만 제가 모른다고 생각하시는 것은 권하지 않습니다.”
```

에드가는 일지를 회수하지 않는다. 관리 프로토콜상 주인공의 가족 기록 열람을 강제로 차단할 권한이 없고, 대시계 점검을 비울 수도 없기 때문이다.

대화 뒤:

- 에드가는 나간다.
- 문은 잠기지 않는다.
- J1 목표는 그대로 유지된다.
- 최초 1회 `MEM_EDGAR_LIBRARY_ENCOUNTER`를 잔류 기억 후보로 기록한다.
- 관계값은 자동 변경하지 않는다.

### 9.9 반복 진입

한 B2 장면에서 에드가 진입은 기본 1회다. 이후 큰 행동은 발소리 잔향만 재생하고 추가 대화를 발생시키지 않는다.

다음 루프:

- 에드가는 이전 만남의 사실을 기억할 수 있다.
- B2 접근 창을 삭제하거나 바꾸지 않는다.
- 표층 인사는 경계도에 따라 한 문장 변형된다.
- J1 완료 뒤에는 B2 전체 압박 이벤트를 다시 요구하지 않는다.

### 9.10 상태

```yaml
event_id: B2
location_id: M1_LIBRARY_INNER
loop_state:
  b2_attention_level: 0
  b2_edgar_entry_used: false
persistent_state:
  b2_hide_discovered: false
  b2_caught_once: false
possible_sub_events:
  - B2_EDGAR_ENTRY
  - B2_HIDE
  - B2_CAUGHT
merge:
  - J1
fail_policy: none
```

## 10. J1: 끝난 뒤 남는 소리

### 10.1 기본 정보

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | `M1_LIBRARY_INNER`, B2 |
| 선행 조건 | 일지 책상 도달 |
| 목표 | 손상된 첫 페이지 복원, 시계망 가설 획득 |
| 플레이 시간 | 5~9분 |
| 실패 | 오배열 LOCAL RETRY |
| 성공 | `journal_stage=1` |
| 다음 | B3-A |

### 10.2 복원 조작

1. 일지 표지를 책상 중앙 홈에 놓는다.
2. 세 장의 얇은 압지 조각을 펼친다.
3. 종이 눌림 자국과 잉크 번짐을 맞춘다.
4. 문장 순서를 배열한다.
5. 마지막 문장 뒤 비어 있는 한 칸과 잉크점을 조사한다.
6. 수첩에 관찰과 가설을 따로 적는다.

압지 조각은 문장 의미만으로도 배열할 수 있고, 종이 결·모서리 홈으로도 확인할 수 있다.

### 10.3 개작 원문

```text
네가 어제를 기억한다면,
얼굴이 가리키는 시각부터 믿지는 마라.

하나는 하루를 세고,
하나는 멈춘 채 떨림을 다음 방으로 넘기며,
하나는 목소리를 잃은 채 마지막 문을 울린다.

틀린 것은 시간이 아니다.
모두 끝났다고 믿은 뒤에도, 한 칸 남아 있는 소리다. ·
```

마지막 마침표와 잉크점 사이에는 글자 한 칸의 빈 공간이 있다.

### 10.4 정보 분리

| 정보 | 상태 |
| --- | --- |
| 세 시계가 기준·중계·출력 역할을 가진다 | `hypothesized` |
| 멈춘 시계도 신호를 전달할 수 있다 | `observed` |
| 소리 없는 시계가 출력일 수 있다 | `hypothesized` |
| 정상 완료 뒤 한 칸이 남는다 | `observed` |
| 위상이 +1이다 | 아직 미표기, B3에서 검증 |

J1은 시계 이름을 정답과 직접 연결하지 않는다.

### 10.5 오답과 힌트

| 상태 | 피드백 |
| --- | --- |
| 문장 순서 오류 | 눌림 자국이 한 줄 어긋남 |
| 압지 앞뒤 오류 | 잉크가 손에 묻지만 문장이 읽히지 않음 |
| 잉크점 미조사 | 완료 가능, B3 힌트 단계에서 재강조 |

힌트:

1. 종이 모서리 홈.
2. `하나는`으로 시작하는 세 문장의 평행 구조.
3. 마지막 줄 오른쪽 여백 확대.

### 10.6 완료 연출

- 기록 시계가 한 번도 울리지 않고 내부 진자만 움직인다.
- 일지 글씨 아래에서 회로 같은 눌림 자국이 한 프레임 보인다.
- 에드가 진입을 겪었다면 문밖 발소리가 J1 마지막 여백에서 멈춘다.

### 10.7 상태

```yaml
event_id: J1
outputs:
  set:
    journal_stage: 1
  add_knowledge:
    - KN_J1_CLOCK_ROLES
    - KN_J1_POST_COMPLETION_GAP
  unlock:
    - LOCK_NORTH_LIBRARY_LINK_INTERACTION
  next_event: B3_A
```

J1 뒤 초상화 패널 안쪽 걸쇠를 조사하면 북쪽 기록 회랑 연결문 숏컷을 해금한다.

## 12. C2: 마라 1 청소 기록

### 12.1 기본 정보

| 항목 | 내용 |

```text
검은 코팅은 때처럼 보이지만 표면 보호층에 가깝다.
강한 약품은 코팅을 벗기고, 그 아래의 가는 선까지 녹였다.

시험선이 가장 오래 남았다.

솔은 쓰지 말 것.
부드러운 천으로 한 번 지나간 자리를 다시 세게 문지르지 말 것.
```

### 12.4 수첩

```text
[관찰] 총량은 8칸이다.
[관찰] 물 = 안정제와 원액의 합 + 2.
[관찰] 강한 세정제와 반복 마찰은 하부 회로를 지운다.
```

`물=5`는 아직 자동 계산하지 않는다.

### 12.5 마라 1 반응 연결

마라 1이 현장에 있으면 기록 위치만 알려 준다.

```text
마라 1: “그 장부 말임다? 제가 썼는데도 제가 읽기 싫게 생겼슴다.
```

### 17.2 복원 조작

1. 아버지 일지의 날짜 조각을 중앙에 놓는다.
2. 연구원 기록의 승인 번호와 색상 서명을 확인한다.
3. `약속`, `전환`, `역할 고정`, `주인공 기동` 순서로 페이지 배열.
4. 획득하지 않은 기록은 회색 인덱스로 남는다.
5. 배열 후 모순 문장을 조사한다.

미획득 기록의 빈칸은 오답 후보가 아니다. BASE 경로에서도 필요한 네 사건 축은 아버지 일지와 시스템 인덱스로 배열할 수 있다.

### 17.4 BASE 개작 원문

```text
나는 그들에게 미래의 삶을 약속했다.
그 약속이 육체를 뜻하는지, 기억의 지속을 뜻하는지
끝까지 분명하게 말하지 않았다.

그들은 집을 움직일 수 있다.
계절을 고치고, 심장을 유지하고, 문을 잠글 수 있다.

하지만 집 밖으로 나가는 문과
누가 이 삶의 주인인지 정하는 권한은 주지 않았다.

그들이 너를 깨운 것은 보호만도, 복수만도 아니었다.
너를 붙잡으면 내가 남긴 약속도 아직 끝나지 않았다고
믿을 수 있었기 때문이다.
```

BASE가 반드시 공개하는 사실:

- 사용인은 연구원 인격이다.
- 아버지는 새 삶의 의미를 불완전하게 설명했다.
- 사용인은 관리 권한만 있고 SUBJECT 권한은 없다.
- 사용인들이 주인공 의식을 강제로 기동했다.
- 코어 절차는 주인공만 완성할 수 있다.

## 18. F1: 아버지의 마지막 기록

### 18.1 기본 정보

| 항목 | 내용 |
| --- | --- |
| 위치 | `H0_CORE_RECORDS` |
| 선행 조건 | F0-E, `subject_authority_restored=true` |
| 목표 | 아버지의 원본 마지막 로그를 주인공이 직접 재생 |
| 플레이 시간 | 9~15분 |
| 실패 | 없음 |
| 관계 | 관계 수와 무관하게 전 세그먼트 제공 |
| 다음 | J5 |

기록은 자동 재생되지 않는다.

```text
[재생한다]
[아직 재생하지 않는다]
```

`아직 재생하지 않는다`를 선택하면 코어 기록실을 더 조사할 수 있다.

### 18.2 인증

재생 전 주인공은 A1에서 만든 자기 표시를 입력한다. 이는 F0-E에서 복원한 SUBJECT 권한을 현재 기록 재생에 사용한다는 확인이다.

```text
CURRENT REQUESTER: SUBJECT
CREATOR LOG: ORIGINAL
EDIT HISTORY: PRESENT
PLAYBACK PERMITTED
```

F1은 J1~J4 페이지가 아버지 기록에서 파생됐지만 일부가 시스템과 사용인에 의해 잘려 나갔음을 인증한다.

### 18.3 세그먼트

| 순서 | 세그먼트 | 필수 사실 |
| ---: | --- | --- |
| 1 | 외부 붕괴 | 지구 환경과 사회 기반 붕괴 |
| 2 | 주인공 냉각 | 생존 가능한 인간 신체 보존 |
| 3 | 낙서 저택 | 주인공의 그림을 고딕 위장 세계로 사용 |
| 4 | 연구원 전환 | 새 육체 약속과 불완전한 동의 |
| 5 | 역할 고정 | 통속의 뇌와 시스템 인격으로 잔류 |
| 6 | 강제 기동 | 아버지 사후 사용인들이 주인공 의식을 깨움 |
| 7 | 미완성 권한 | 현실·잔류 어느 쪽도 대신 결정하지 않음 |

플레이어는 순서대로 듣되 각 세그먼트 사이에서 관련 자료를 재열람할 수 있다. 최초 재생에서 필수 세그먼트를 완전히 건너뛸 수는 없고, 반복 재생은 자유롭게 건너뛴다.

### 18.5 마라 2 로그

F1은 관계 상태와 무관하게 기술 사실을 공개한다.

- 아버지가 마라 2에게 아카이브 우선순위를 맡겼다.
- 자동 삭제 정책을 완전히 해제하지 못했다.
- 마라 2가 다른 연구원 원본을 보존하려 자기 영역을 양도했다.
- 아버지는 이를 중단할 권한을 공유하지 않았다.

E3_5 완료 시:

- 병합 또는 분리 보존 결과를 감정 주석으로 추가.

E3_5 미완료 시:

- 보라 채널 소유자 이름은 P3B 기준으로 표시.
- 자기 희생 동기는 로그 추론으로만 제공.
- 마라 2 개인 고백 음성은 재생하지 않는다.

### 18.6 감각

- 아버지 음성은 방 안이 아니라 냉각관 내부에서 들리는 듯 울린다.
- 낙서 저택 세그먼트에서 종이 냄새가 나지만 화면에는 서버 랙이 보인다.
- `감옥`이라는 단어에서 고딕 벽지가 한 프레임만 돌아온다.
- 마지막 문장 뒤 기계음은 멈추지만 주인공의 필기음만 남는다.

### 19.2 페이지

F1 재생 장치 아래에서 마지막 종이 페이지가 출력된다. 마지막 두 줄은 비어 있다.

개작 원문:

```text
나는 너를 살린다는 이유로 네 시간을 멈췄다.
그들을 구한다는 말로 그들의 시간을 역할 속에 묶었다.

밖은 아직 안전하다고 약속할 수 없다.
안은 거짓에서 시작됐지만, 네가 고른다면 전부 거짓인 삶은 아닐 수 있다.

나는 용서를 요구하지 않는다.
무엇을 선택하라고 남길 권리도 없다.

마지막 두 줄은 비워 둔다.
어디로 갈지를 쓰기 전에,
누가 이 문장을 쓰고 있는지 먼저 확인해라.
```

### 19.3 상호작용

주인공은 기존 문장을 고르지 않고 직접 짧은 문장을 쓴다.

필수 의미:

```text
이 문장은 지금의 내가 쓴다.
마지막 결정은 아직 확정하지 않는다.
```

허용 표현은 여러 개지만 의미 태그는 동일하다.

잘못된 입력:

| 입력 | 피드백 |
| --- | --- |
| 아버지 문장 불러오기 | CREATOR 권한, 현재 주체 아님 |
| 시스템 자동 완성 | SYSTEM 권한 |
| 사용인 기록 붙여넣기 | RESIDENT 또는 CUSTODIAN 권한 |
| 현실·잔류를 여기서 확정 | `FINAL DECISION은 코어실에서만 확정` |

J5는 F0-E의 `f0_provisional_intent`를 읽거나 바꾸지 않는다.

### 19.4 출력

```text
CURRENT AUTHOR: SUBJECT
JOURNAL CHAIN: AUTHENTICATED
FINAL DECISION: UNSET
```

### 19.5 역할

- J4는 과거 사건의 정보량을 정리한다.
- F1은 아버지 기록과 수정 이력을 인증한다.
- J5는 주인공이 현재 문장을 작성할 권한을 인증한다.
- EDC는 현실 또는 잔류를 실제로 확정한다.

### 19.6 상태

```yaml
event_id: J5
location_id: H0_CORE_RECORDS
outputs:
  set:
    journal_stage: 5
  authenticate:
    - JOURNAL_CHAIN
    - CURRENT_SUBJECT_AUTHORSHIP
  assert:
    final_decision: unset
  forbidden_effects:
    - final_decision
    - f0_provisional_intent
    - relationship_changes
  next_event: F2
```

- 정보가 추가될수록 방의 정체보다 주인공의 감각 신뢰가 흔들리게 한다.
- 같은 문장을 반복할 때 음성 속도보다 종이 질감과 공간 소리를 바꾼다.
- 에드가 압박은 실패 공포가 아니라 알고도 모르는 척하는 사람과 한 방에 있는 불안을 목표로 한다.
- 아버지 기록은 따뜻한 회상보다 책임 회피와 뒤늦은 자기 인식이 먼저 들리게 한다.

## 24. 접근성

- 모든 긴 기록은 자막, 기록 로그와 재생 속도 조절을 제공한다.
- 방향성 소리는 시각 방향선과 공간명 자막으로 대체한다.
- B2 에드가 진입은 소리 외 문손잡이·화면 가장자리 텍스트로 예고한다.
- B2 숨기에는 시간 제한, QTE, 빠른 클릭을 사용하지 않는다.
- 색 제거 모드에서도 이름·문양·선 패턴으로 C5와 기록 출처를 판독한다.
- 드래그 대신 선택·회전 버튼으로 모든 페이지 배열과 중첩을 수행한다.
- 글리치 강도 0에서도 핵심 이질감은 독백과 진단 문구로 남는다.
- 긴 문장을 요약하더라도 퍼즐에 필요한 수치·순서·부정문을 생략하지 않는다.

## 26. 소프트락 방지

- 마라 2 시간표 없이 B1 해결 가능.
- B2 점검 벽감을 발견하지 않아도 발각 대화로 진행.
- 에드가 진입 여부와 무관하게 J1 접근 가능.
- 에드가 alert 5에서도 B2 잠금 없음.
- J1 잉크점을 놓쳐도 B3 단계형 힌트로 재발견.
- C2·C2-1 기록은 사용인이 부재해도 오브젝트에서 획득.
- C5 짧은 반응을 건너뛰어도 J3 진행.
- 연구원 기록 0개여도 J4 BASE 완료.
- E3_5 미완료여도 F1 기술 로그와 익명 보라 인덱스 제공.
- F1 재생을 미뤄도 방 안 조사는 가능하지만 F2는 J5 전까지 잠김.
- J5에서 현실·잔류를 선택하도록 강제하지 않음.

## 27. QA 시나리오

### 27.1 B1

1. 에드가·루카 단서만으로 접근 창 추론.
2. 에드가·마라 1 단서만으로 접근 창 추론.
3. 마라 2 보조 단서 없이 완료.
4. 모든 오답 시간대를 선택하고 관계값 불변 확인.

### 27.2 B2

| 경로 | 기대 |
| --- | --- |
| 조용히 일지만 조사 | 에드가 미진입 또는 최소 예고, J1 |
| 주의도 상승 후 숨기 | B2_HIDE 뒤 J1 |
| 주의도 상승 후 남기 | B2_CAUGHT 대화 뒤 J1 |

| EDGAR.alert=5 | 경고 대사, 진행 유지 |
| 반복 루프 | 첫 대화 축약, B2 접근 창 유지 |

### 27.3 일지

1. J1~J3 오배열 후 LOCAL RETRY.
2. J1 잉크점 미조사 상태에서 B3 힌트 확인.
3. J2가 C3 정답 수치를 직접 노출하지 않는지 확인.
4. J3가 D0-A 정확한 변환을 직접 말하지 않는지 확인.
5. J4 0·1·2·4·5 기록 변형 확인.
6. J4 뒤 E3_4 미완료 시 E3_4M 진입 확인.

### 27.4 F1·J5

1. 관계 0명으로 F1 전체 세그먼트 재생.
2. E3_5 미완료 시 마라 2 기술 로그 제공.
3. J5 외부 권한 문장 사용 시 LOCAL RETRY.
4. J5 완료 뒤 `final_decision=unset` 확인.
5. F0 provisional intent 3종이 J5에서 변하지 않는지 확인.

### 27.5 접근성

1. 음량 0으로 B1·B2·J1 완료.
2. 색 제거 모드로 C5 채널 판독.
3. 드래그 없이 모든 페이지 배열.
4. 모션 감소로 B2 숨기와 C5 연출 완료.
5. 글리치 0에서 필수 정보 누락 없음.

## 28. 완료 기준

- P3 외부 서고와 B2 기록 내실의 기능이 분리된다.
- B1은 핵심 단서 2개와 선택 보조 단서 구조를 가진다.
- B2에는 비실패 에드가 대화와 점검 벽감 숨기 경로가 있다.
- B2의 모든 분기가 J1로 합류한다.
- J1~J5 원문과 복원 조작이 정의된다.
- C2와 C2-1의 단서가 C3 정답을 유일하게 도출할 수 있다.
- C5 정보와 J3·D0의 책임이 D0-A 퍼즐과 분리된다.
- J4가 E3_4/E3_4M보다 먼저 실행된다.
- J4 기록 수가 메인 진행을 차단하지 않는다.
- F1과 J5가 관계·엔딩 의향을 강제로 바꾸지 않는다.
- 모든 주요 정보가 knowledge_id와 수명 주기 상태를 가진다.
- 색·음향·드래그·빠른 입력 없이 필수 조사를 완료할 수 있다.
