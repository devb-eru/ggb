# GGB v0.4 이벤트 상세 02: 루프·영구 정보·숏컷

## 1. RESET

정상 처리 순서:

1. 수첩·일지·관계·색상 서명 확정.
2. 사용인 직접 관찰을 잔류 기억 로그로 압축.
3. 자동 저장.
4. 물리·당일 상태 초기화.
5. 챕터 기준 글리치 재적용.
6. 침실 아침 로드.
7. MORNING과 ROUTE 실행.

```yaml
event_id: RESET
persist:
  - notebook
  - journal
  - knowledge
  - failure_logs
  - relationships
  - servant_residual_memory
  - color_signatures_known
clear:
  - inventory
  - physical_color_residue
  - puzzle_physical_state
  - npc_positions
  - daily_interference
```

## 2. SYS-01 첫 리셋

- 에드가가 같은 억양으로 인사.
- 같은 새가 같은 궤도로 이동.
- P3B 초상화 이름표와 물리 배치는 원래대로 복원.
- 수첩 기록은 유지.
- 마라 2의 보라색 이름표 가장자리가 한 프레임 늦게 정렬.

주인공이 창문, 수첩, 초상화 기록 중 하나를 비교하면 `loop_awareness = suspected`.

## 3. A1·A2 수첩 표시

### A1

표시 선택:

- `내일 아침, 이 문장을 읽어.`
- 어린 시절 집 도형.
- 페이지 모서리 잉크 얼룩.

표시는 `self_authored_mark`로 영구 저장한다.

### A2

다음 아침 표시가 남은 것을 확인한다.

- `loop_awareness = confirmed`.
- 수첩 영구 기록·실패 로그 탭 해금.
- F0-E의 과거 자기 인증 원자료가 됨.

마라 2에게 보여주면:

```text
마라 2:
이건 내 글씨도, 시스템 글씨도 아니네!
네가 쓴 거야? 제법인데!
```

정답 의미는 후반까지 설명하지 않는다.

## 4. 색상 서명 리셋 규칙

| 요소 | 리셋 |
| --- | --- |
| 초상화에 묻힌 색 얼룩 | 초기화 |
| 의상 기본색 | 챕터 기준 복원 |
| 수첩의 색·문양 기록 | 유지 |
| 인격 로그의 소유자 서명 | 유지 |
| 당일 글리치 잔상 | 초기화 |
| D5 이후 분리된 색 채널 | 챕터 상태 유지 |

색 잔상이 남았다는 이유만으로 사용인의 정확한 기억 내용을 알려주지 않는다.

## 5. ROUTE

```text
BROKEN_RESET
> J3+D1 실패
> J3
> J2+C4 실패
> J2
> J1+B3 실패
> J1
> A1 표시 미확인
> A1
```

P3B와 마라 2 관계 이벤트는 ROUTE 필수 우선순위에 포함하지 않는다.

## 6. SYS-02 일과 압축

| 완료 횟수 | 방식 |
| --- | --- |
| 최초 | 전체 플레이 |
| 2회 | 상호작용 절반 |
| 3회 이상 | `익숙한 순서대로 마친다` |

P3B 숏컷:

1. 주인공이 이름표 선반을 연다.
2. 다섯 문양이 한 번씩 점등.
3. 마라 2가 `이번엔 안 틀렸네! 재미없어!`라고 말함.
4. 5~12초 안에 완료.

새 색 이상이나 MARA2_S1이 있으면 숏컷을 중단하고 해당 이벤트를 재생한다.

## 7. BSHORT

유지:

- B3-A 탁본 조립.
- 검증된 기준·중계·출력 역할.
- 실패한 위상.

실행:

- 아침 일과 압축.
- 시간표대로 저녁까지 전진.
- 서쪽 대시계 입구에서 조작권 반환.
- B3-B의 미확정 입력만 재시도.

## 8. CSHORT

유지:

- C3 제약식 `물 5·안정제 1·원액 2`.
- C4 파형 중첩 가설.
- 실패한 경로와 에드가 개입 정보.

초기화:

- 세정제와 천.
- 거울 코팅.
- NPC 위치.

실행:

- 재료 빠른 확보.
- 실제 제조 입력은 수행.
- C4 마른 천 시험과 실제 닦기 재시도.

## 9. DSHORT

유지:

- D0-A 도면 중첩.
- 축 깊이 `2·1·3`.
- 검증된 축 단계.

실행:

- 서재 단서 재독은 요약.
- 지하 입구까지 빠른 동선.
- 미확정 축부터 실제 입력.

## 10. SYS-03 반복 대화

우선순위:

1. 현재 메인 이벤트.
2. 처음 보는 관계 이벤트.
3. 잔류 기억 반응.
4. 색상 서명 이상.
5. 반복 요약.
6. 기본 역할 대사.

마라 2 반복 단계:

| 횟수 | 반응 |
| --- | --- |
| 1 | 장난스러운 전체 설명 |
| 2 | `그건 어제도 말했잖아!`라고 했다가 멈춤 |
| 3+ | 짧은 놀림 뒤 핵심 한 문장 |
| 기억 균열 | 자기 이름을 잘못 말했다가 즉시 정정 |

## 11. SYS-04 목표 복구

힌트 출처:

- 환경.
- 수첩.
- 주인공 독백.
- 선택형 사용인 대화.

관계가 낮아도 환경과 수첩으로 필수 정보를 제공한다.

## 12. D5 이후

| 상태 | 잠의 기능 |
| --- | --- |
| FRACTURE_SLEEP | 정상 리셋 시도 실패 |
| BROKEN_RESET | 다른 아침 |
| POST_BROKEN_REST | 휴식·시간 전환·기억 동기화 |
| FINAL_SLEEP_LOCK | 최종 선택 회피 불가 |

D5 이후 물리 색 채널은 초기화되지 않으며 휴식 때 현재 관계 상태에 맞춰 동기화된다.

## 13. 소프트락 방지

- P3B 미완료는 프롤로그 일과 게이트에서 명시.
- 색을 못 구분하면 자동 패턴 모드 제안.
- 숏컷 중 새 이벤트 발생 시 압축 중단.
- 관계 이벤트는 메인 ROUTE를 변경하지 않음.
- 세 번째 하드 실패부터 검증된 퍼즐 하위 단계를 고정 가능.
