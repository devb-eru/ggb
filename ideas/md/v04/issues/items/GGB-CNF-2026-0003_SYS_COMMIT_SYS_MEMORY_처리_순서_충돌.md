## 문제

`02`는 사용인 기억 압축 뒤 영구 상태를 커밋한다. `07`의 공식 체인은 다음 순서다.

```text
NORMAL_SLEEP
→ SYS_COMMIT
→ SYS_MEMORY
→ NORMAL_RESET
→ MORNING
→ ROUTE
```

## 권장 해결

`07`을 정본으로 채택하고 두 커밋 성공 뒤에만 물리 상태를 폐기한다.

## 해결 체크리스트

- [ ] `02` 처리 순서 교정.
- [ ] `17`에 단계별 재개 상태 추가.
- [ ] 동일 transaction ID의 중복 적용 방지.
- [ ] SYS_COMMIT·SYS_MEMORY 각각의 실패 복구 검증.

## 해결 증거

아직 없음.
