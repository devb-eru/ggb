## 문제

상세 문서는 `failure_record`, `active/resolved/superseded`, `route_snapshot`, `shortcut_resume`을 사용하지만 구현 계약에는 단일 `hard_failure_id`만 있다.

## 권장 해결

- 영구 실패 기록과 현재 루프 물리 실패 상태를 분리한다.
- B/C/D 숏컷마다 검증된 단계와 미확정 입력을 저장한다.

## 해결 체크리스트

- [ ] failure record 배열 또는 ID 맵 정의.
- [ ] active→resolved 전환 정의.
- [ ] BSHORT·CSHORT·DSHORT 재개 데이터 정의.
- [ ] 성공 뒤 과거 active 기록이 ROUTE를 다시 열지 않는지 확인.

## 해결 증거

아직 없음.
