## 문제

`02~04`는 S3 생성을 최초 BROKEN_RESET에만 수행하고 이후 수면은 `POST_BROKEN_REST`로 현재 수리 상태를 유지한다. `17`은 `broken_reset_triggered=true`일 때마다 S3 기준 `loop_state`를 재생성하는 구조다.

## 권장 해결

```text
NORMAL_LOOP
→ FRACTURE_SLEEP
→ BROKEN_RESET_ONCE
→ POST_BROKEN
→ POST_BROKEN_REST 반복
```

## 해결 체크리스트

- [ ] S3 생성은 `BROKEN_RESET_ONCE`에서만 실행.
- [ ] `POST_BROKEN_REST`는 시간·피로만 전환.
- [ ] E3 수리 상태와 공간 상태 유지.
- [ ] F3 이후 `FINAL_SLEEP_LOCK` 확인.

## 해결 증거

아직 없음.
