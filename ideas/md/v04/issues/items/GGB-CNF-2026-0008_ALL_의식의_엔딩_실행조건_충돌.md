## 문제

상위 관계 규칙은 사용인 관계 상태가 F0, EDC, 현실·잔류 엔딩을 차단하지 않는다고 정한다. 그러나 상세화 전 엔딩 문서는 ALL 장면에서 주인공이 선언문을 다시 써야 최종 엔딩을 실행할 수 있다고 서술했다.

이 해석은 다음 문제를 만든다.

- 다섯 명 전원 완료 플레이가 오히려 추가 입력 게이트를 얻는다.
- 필기 입력 실패·접근성 대체 입력·중단 재개의 처리에 따라 확정한 엔딩이 멈출 수 있다.
- `all_servants_complete`가 반응 합성 조건을 넘어 `final_decision` 생성 조건처럼 작동한다.

## 채택 해결안

`ED_ALL_CEREMONY`를 **EDC 원자 커밋 뒤, 선택한 엔딩 본문 전**의 조건부 비차단 장면으로 고정한다.

```text
EDC_FINAL_DECISION
→ final_decision·final_choice_relation 원자 커밋
→ all_servants_complete 검사
→ true: ED_ALL_CEREMONY
→ false 또는 의식 완료: 선택한 엔딩 본문
```

- 의식은 `auto_dispatch_when_guard=true`인 조건 장면이다.
- `required_for_ending_success=false`이며 엔딩 선택·성공·관계 수치를 쓰지 않는다.
- 필기 추적은 이미 확정된 선택을 확인하는 의례다.
- 오답과 시간 제한을 두지 않고 자동 추적 대체 입력을 제공한다.
- 중단 시 확정 분기를 유지하고 첫 미완료 인증부터 재개한다.

## 해결 체크리스트

- [x] EDC가 ALL 검사 전에 최종 선택을 원자 커밋하도록 배치.
- [x] 의식 입력을 엔딩 허가 암호에서 확정 후 서명으로 변경.
- [x] 금지 효과에 `final_decision`, `final_choice_relation`, `ending_success`, 관계 변화를 명시.
- [x] 접근성 자동 추적과 중단 재개 규칙 정의.
- [x] 현실·잔류 양쪽이 동일한 비차단 계약을 사용함을 확인.

## 회귀 방지 규칙

1. `all_servants_complete`를 EDC 선택지 guard에 사용하지 않는다.
2. ALL 전용 입력은 `final_decision`이나 엔딩 성공 플래그를 쓰지 않는다.
3. ALL 장면을 추가할 때 현실·잔류 공통으로 중단 재개와 접근성 대체 입력을 제공한다.
