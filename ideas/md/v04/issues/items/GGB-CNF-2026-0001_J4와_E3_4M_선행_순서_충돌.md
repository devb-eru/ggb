## 문제

`03`과 `04`는 `J4 → E3_4 완료 검사 → E3_4M → E5` 순서를 사용한다. `13`은 J4 선행 조건으로 `E3_4 또는 E3_4M`을 요구한다.

```text
J4에 E3_4M 필요
→ E3_4M에 J4 필요
→ 관계 이벤트 0명 최소 경로 정지
```

## 권장 해결

```text
E_HUB 조사 종료
→ J4
→ E3_4_complete 검사
→ false이면 E3_4M
→ E5
```

## 해결 체크리스트

- [x] `13`의 J4 선행 조건에서 E3_4/E3_4M 제거.
- [x] `12`의 E3_4M 선행 조건을 `J4 완료 && E3_4 미완료`로 변경.
- [x] E5 조건을 `J4 && (E3_4 || E3_4M)`으로 통일.
- [x] 관계 이벤트 0명 경로 검증.

## 해결 증거

- `13`: J4 선행 조건을 `E2_INTRO, E_HUB 조사 종료`로 변경.
- `12`: E3_4M 선행 조건을 `J4 완료, E3_4 미완료, E5 진입 전`으로 변경.
- `03`, `04`: `J4 → CHK_EDGAR_CORE → E3_4M 또는 관계 집계 → E5` 순서 유지 확인.
- `rg` 검사로 구식 J4 선행 조건이 남지 않았음을 확인.
