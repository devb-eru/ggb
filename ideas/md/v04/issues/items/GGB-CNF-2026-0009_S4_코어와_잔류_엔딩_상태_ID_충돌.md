## 문제

기존 시스템 문서는 F구간 코어 진입 상태를 `world_phase=S4`로 정의한다. 엔딩 상세 초안은 잔류 선택 뒤 안정화된 파열 상태에도 같은 `S4_STABILIZED_FRACTURE`를 사용했다.

하나의 상태 ID가 서로 다른 장면 생애주기를 뜻하면 다음 문제가 발생한다.

- F0 코어 진입만으로 잔류 엔딩 전용 오버레이와 오브젝트 반응이 활성화될 수 있다.
- 저장 재개 시 코어 퍼즐과 잔류 에필로그 중 어느 라우터를 복원해야 하는지 결정할 수 없다.
- `world_phase=S4`만 기록한 구 데이터는 최종 선택 없이 잔류 엔딩으로 오인될 수 있다.
- 공간, 접근성, 색상 경계, 엔딩 완료 판정이 같은 상태를 서로 다른 뜻으로 소비한다.

## 채택 해결안

선행 문서의 의미를 우선해 `S4`는 F구간 코어 상태로 보존하고, 잔류 엔딩 전용 상태 `S5_STABILIZED_FRACTURE`를 신설한다.

```text
S3_POST_BROKEN
→ S4 (코어)
→ EDC_FINAL_DECISION
   ├─ reality: R0_* 현실 장면
   └─ stay: S5 (S5_STABILIZED_FRACTURE)
```

- `world_phase=S4`는 F0~F3과 EDC 전 상태만 담당한다.
- `world_phase=S5`는 `final_decision=stay`가 원자 커밋된 뒤 `EDS_STABILIZE`에서만 진입한다. `S5_STABILIZED_FRACTURE`는 이 단계의 설계 상태명이다.
- S5는 새 저택을 생성하지 않고 S3 시설 프레임과 기존 `M1_CENTRAL_HALL`, `M1_DINING_ROOM`에 안정화 오버레이를 적용한다.
- 현실 엔딩은 저택의 `world_phase`를 변경하지 않고 `R0_*` 현실 장면으로 전환한다.
- 구 저장값 `S4_STABILIZED_FRACTURE`는 `final_decision=stay` 또는 `EDS_*` 진행 이력이 있을 때만 S5로 이관한다. 증거가 없으면 S4 코어로 유지한다.

## 회귀 방지 규칙

1. `world_phase=S4`를 잔류 엔딩 렌더링 조건으로 사용하지 않는다.
2. `S5_STABILIZED_FRACTURE` 진입은 `final_decision=stay` 원자 커밋 뒤에만 허용한다.
3. 구 저장을 S5로 이관할 때 상태 문자열 하나만 보지 않고 최종 선택 또는 EDS 진행 증거를 함께 확인한다.
4. 현실 엔딩 공간은 저택 `world_phase`가 아니라 `R0_*` location과 ending branch로 판정한다.
