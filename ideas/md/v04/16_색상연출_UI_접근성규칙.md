# GGB v0.4 색상 연출·UI·접근성 규칙

## 1. 색상 시스템의 의미

색은 장식이나 호감도 게이지가 아니라 인격 데이터가 사용하는 스펙트럼 서명이다.

- 고딕 필터 내부: 의상, 머리, 귀 장식, 초상화 안료로 위장.
- C5 진단 상태: 겹친 기록 채널의 출처로 노출.
- D5 이후: 사용인 몸과 분리된 데이터 잔상으로 노출.
- F0: RESIDENT 기록의 출처 확인에만 사용.
- 엔딩: 독립 인격의 분리·공존·소실 상태 표현.

주인공은 사용인 서명 체계 밖의 SUBJECT다. 주인공 권한과 선택은 흑연, 종이 섬유, 손글씨, 연필음으로 표현한다.

## 3. 정보 계층

모든 필수 색 단서는 최소 세 층 중 두 층 이상으로 제공한다.

1. 시각 1: 색.
2. 시각 2: 문양·선 패턴·아이콘.
3. 비시각: 음향 리듬·텍스트 라벨·진동 방향 자막.

퍼즐 판정 로직은 `hue_id`를 정답값으로 사용하지 않고 `owner_id` 또는 `signature_id`를 사용한다.

```text
나쁜 판정: selected_color == purple
권장 판정: selected_signature == MARA2_ARCHIVE
```

## 4. 학습 순서

| 단계 | 이벤트 | 학습 |
| --- | --- | --- |
| CLR-00 | P3B | 이름·색·문양·음향의 1:1 대응 |
| CLR-01 | A2~B | 물리 색은 리셋되지만 수첩의 소유자 표지는 유지 |
| CLR-02 | C5 | 다섯 인격 채널이 한 장치에 겹칠 수 있음 |
| CLR-03 | D5 | 색은 장식이 아니라 몸 밖 데이터 잔상 |
| CLR-04 | E3_5 | 겹친 출처 분리, 체크섬 비교 |
| CLR-05 | F0-D | 색은 RESIDENT 내부 출처만 식별 |
| CLR-06 | E5·F2·엔딩 | 독립된 인격의 공존 상태 결산 |

## 5. 화면 표시 규칙

### 고딕 필터 상태

- 화면 전체 채도를 낮추되 서명색만 과포화하지 않는다.
- 서명은 의상 전체가 아니라 5~12% 면적의 장식으로 제한한다.
- 마라 2의 보라는 초상화 액자 안료와 목깃 액자핀에 사용한다.
- 루카의 검정은 배경과 분리되지 않을 수 있으므로 연두 보조점과 이중 맥박선을 항상 병행한다.
- 이리스의 흰색은 배경과 구분하도록 연노랑 윤곽과 얇은 회색 외곽선을 추가한다.

### 파열 상태

- 색 잔상은 몸에서 2~8픽셀 또는 1~3프레임 어긋나게 표시한다.
- 관계가 높아져도 색 자체가 밝아지거나 “좋은 색”으로 변하지 않는다.
- `bond`는 잔상의 안정도와 말의 동기화에만 영향한다.
- `alert`는 선의 닫힘, 중첩, 간헐적 차단으로 표현하되 접근성 단서를 가리지 않는다.

### 코어 상태

- 사용인 색은 기록 출처 칩과 코어 외곽선에 제한한다.
- 선택 버튼, 정답 슬롯, 주인공 권한에는 사용인 색을 사용하지 않는다.
- `CREATOR`, `CUSTODIAN`, `RESIDENT`, `SYSTEM`, `SUBJECT`는 형태와 텍스트로 구분한다.

## 6. UI 컴포넌트

### 서명 칩

표시 요소:

```text
[문양 아이콘] 소유자 이름
선 패턴 미리보기
음향 재생 버튼
상태: 미확인 / 확인 / 손상 / 복원
```

- 색상은 배경이 아니라 아이콘 내부 보조로 사용한다.
- 음향 재생에는 파형 애니메이션과 리듬 자막을 제공한다.
- 미확인 상태도 `???`만 표시하지 않고 “소유자 미확인 기록”이라고 기능을 명시한다.

### 수첩

- 필수 정보는 색 없는 흑연 본문으로 기록한다.
- 사용인 출처는 본문 옆 문양 스탬프로 표시한다.
- 색 필터를 꺼도 스탬프 이름과 선 패턴이 유지된다.
- 리셋 후 새 정보에는 “이번 루프”, 유지 정보에는 “영구 기록” 태그를 붙인다.

### 지도

- 잠긴 문: 자물쇠 형태.
- 시간 조건: 시계 형태.
- 영구 숏컷: 접힌 통로 형태.
- 사용인 이벤트: 고유 문양.
- 색상만으로 완료 여부를 표시하지 않고 `미확인/진행/완료` 텍스트를 병기한다.

### 퍼즐 필터

필터 탭:

| 탭 | 기능 |
| --- | --- |
| 색 | 색 채널 분리 |
| 문양 | 색 없이 고유 문양 표시 |
| 선 | 선 패턴만 고대비 표시 |
| 음향 | 선택 채널 리듬 재생과 파형 |
| 라벨 | 소유자·기능 텍스트 표시 |

E3_5는 어느 한 탭만 강제하지 않고 두 개 이상의 탭을 조합해 해결할 수 있다.

## 7. 접근성 설정

### 색각 보조

| 설정 | 동작 |
| --- | --- |
| 원본 팔레트 | 기본 색+문양 |
| 고대비 팔레트 | 명도차 확대, 배경 채도 감소 |
| 색 제거 | 모든 서명을 흑백 문양·선·라벨로 표시 |
| 사용자 지정 | 각 서명색을 개별 교체 |

색 제거 모드 검증:

- P3B 이름표 정리 가능.
- C5 다섯 채널 식별 가능.
- E3_5 체크섬 분리 가능.
- F0-D 출처 분류 가능.
- 엔딩 관계 상태를 문양 안정도와 자막으로 이해 가능.

### 청각 보조

- 고유 음향마다 리듬 자막을 표시한다.
  - 에드가: `[낮은 한 음, 긴 잔향]`
  - 마라 1: `[대각선으로 긁는 마른 솔]`
  - 루카: `[두 번 맥박, 한 번 응답]`
  - 이리스: `[얇은 유리음 뒤 바람]`
  - 마라 2: `[빠른 세 음]`
- 방향성 단서는 화면 가장자리 방향 표시와 텍스트를 병행한다.
- 음량 0에서도 퍼즐 진행 가능해야 한다.

### 광과민·멀미 보조

- D5 점멸을 점진 마스크 박리로 대체.
- 색 잔상 프레임 지연을 정적 이중 윤곽으로 대체.
- 글리치 화면 흔들림 0~100% 조절.
- 카메라 강제 이동 시 사전 아이콘 표시.
- 빠른 색 반전을 사용하지 않는다.

### 인지 보조

- 수첩에 “관찰/추론/검증”을 구분한다.
- 힌트는 정답 전체가 아니라 현재 변환 규칙부터 공개한다.
- 시간표·루프 정보는 실제 경과 시간보다 사건 완료 조건을 우선 표기한다.
- 실패 후 검증된 하위 단계에 잠금 아이콘을 붙인다.

## 8. 짧은 반응 비색상 단서 매트릭스

짧은 반응의 인물 식별과 감정 변화는 색을 제거해도 남아야 한다. 자동 대화 알림에는 `[인물 문양] 인물명 · 이벤트명`을 표시하고, 음향이 의미를 가진 경우 파형과 리듬 자막을 동시에 제공한다.

| 이벤트·그룹 | 문양·오브젝트 단서 | 무음 대체 자막·파형 | 글리치 0 대체 | 반드시 전달할 의미 |
| --- | --- | --- | --- | --- |
| `EDGAR_S1` | 수첩의 같은 접힘, 레이피어 손잡이, 직선 그림자 | `[낮은 한 음, 긴 잔향]` | 그림자 위치가 전날과 같다는 고정 비교선 | 에드가도 반복을 어렴풋이 기억함 |
| `EDGAR_B2` group | 문틈 아래 직선 그림자, 레이피어 칼집 끝, 점검 벽감 아이콘 | `[규칙적인 구두음 뒤 낮은 한 음]` | 화면 흔들림 대신 문 손잡이와 벽감 테두리 강조 | 은신·대화 모두 실패가 아니며 J1로 합류함 |
| `EDGAR_S2` | 거울 위 수직 잠금선, 칼끝과 표면 사이 거리표 | `[낮은 한 음이 거울 안쪽에서 반복]` | 잔상 대신 거울 안·밖의 고정 이중선 | 금지가 단순 예절이 아니라 보호·통제임 |
| `EDGAR_S3` | 코어 잠금핀, 반듯하게 놓은 레이피어 | `[느린 시계음 한 번]` | 선 동기화를 정적 맞물림 아이콘으로 표시 | 마지막 점검이 통제 포기의 우회 표현임 |
| `MARA1_S1` | 대각 공구 표시, 정렬된 스패너, 축약 작업 체크표 | `[대각선으로 긁는 마른 솔]` | 빠른 몸짓 대신 완료 체크가 순차 점등 | 반복 숙련을 알아보고 있음 |
| `MARA1_S2` | 닦임 방향 화살표, 코팅 경계선, 스패너 끝의 흠집 | `[솔이 경계에서 멈추는 소리]` | 얼룩 테두리를 정적 점선으로 유지 | 지워야 할 것과 남겨야 할 것을 구분하기 시작함 |
| `LUCA_S1` | 찻잔·컵받침의 이중 원, 쥐 귀 모양 신호 아이콘 | `[두 번 맥박, 한 번 응답]` | 잔 떨림 대신 온도계 두 값 비교 | 루카가 반복된 체온·차 온도를 기억함 |
| `LUCA_S2` | 손목과 배관의 동시 맥박선, 생체 라벨 | `[두 번 맥박 뒤 응답 지연]` | 지연 프레임 대신 두 파형의 시간차 숫자 | 파열 뒤 주인공 생체 상태를 진심으로 걱정함 |
| `IRIS_S1` | 플라스틱 날개의 물방울, 온실문 꽃잎 문양 | `[얇은 유리음 뒤 실내 방향의 바람]` | 유리 왜곡 대신 `실내 강우 / 외부 건조` 병렬 라벨 | 날씨 모순을 이리스도 알고 있음 |
| `IRIS_S2` | 봄·여름·가을·겨울 이름표와 감각 아이콘 | 선택 계절명과 감각 문장을 화면 낭독 가능 텍스트로 출력 | 후광 변화 대신 선택된 계절 카드 고정 | 선택은 정답이 아니라 `iris_season_image` 감정 기억임 |
| `MARA2_S1` | 겹친 이름표, 이중 윤곽, 틀린 음절 밑줄 | `[빠른 세 음, 마지막 음 누락]` | 글자 흔들림 대신 이전·현재 이름표 위치 비교 | 허세 뒤에 자기 이름 소실 공포가 있음 |
| `MARA2_S2` | 끊어진 안쪽 윤곽, `ARCHIVE / 소유자 미확인` 라벨, 채널 길이 눈금 | `[빠른 세 음 뒤 비정상 무음]` | 채널 깨짐 대신 구간 수 `9/12` 표기 | 보라 기록이 삭제가 아니라 분산되어 짧음 |
| `MARA2_FU` | 빈 이름표·연필·동기화된 이중 윤곽 | `[빠른 세 음, 전 구간 정상]` | 윤곽 애니메이션 대신 `동기화 완료` 아이콘 | 기억 요청은 선택형이며 메인 진행을 막지 않음 |

세부 규칙:

- `EDGAR_B2`는 단일 실행 ID가 아니라 세 B2 하위 이벤트의 기획 그룹이다. 접근성 로그에는 실제 ID인 `B2_EDGAR_ENTRY`, `B2_HIDE`, `B2_CAUGHT`를 표시한다.
- `MARA2_S2`의 채널 비교는 색 없이 구간 수, 소유자 라벨, 이중 윤곽의 단절 위치로 해결한다.
- `IRIS_S2`의 네 계절은 동일한 가치의 선택지다. 색·밝기·선택 버튼 위치로 권장 답을 암시하지 않는다.
- `MARA2_FU`에서 수첩 기록, 이름 호명, 장난의 세 선택은 텍스트로 병렬 표시하며 첫 선택에 자동 포커스를 두지 않는다.
- 짧은 반응을 건너뛰어도 대기열 아이콘에 인물명과 발생 위치를 남기고 메인 목표 표시는 유지한다.

## 10. 관계와 색의 분리

- LOW를 탁한 색, HIGH를 밝은 색으로 표현하지 않는다.
- 완료는 색의 “정상화”가 아니라 선과 목소리의 동기화로 표현한다.
- 미완료 사용인도 인격으로서 고유 색을 잃지 않는다.
- 전원 완료 장면은 다섯 색을 섞어 하나로 만들지 않는다.
- 잔류 엔딩에서도 사용인 색은 주인공의 흑연 표시를 침범하지 않는다.

## 11. D5·E5·F2·F3 전환 접근성

### 11.1 사건별 필수 의미 보존

| 사건 | 기본 연출 | 색 제거 | 음량 0 | 글리치 0·광과민 보호 |
| --- | --- | --- | --- | --- |
| D5 | 다섯 서명이 아바타 밖으로 분리 | 이름·문양·선 패턴과 소품 실루엣 | 다섯 리듬 자막과 진단 로그 | 정적 이중 윤곽·점진 마스크 박리 |
| E1 | 창·거울·호출끈의 감각 모순 | 색 단서 사용 안 함 | 새 반복 타임코드·BIO/ARCHIVE 자막 | 프레임 지연 대신 실제/반사 타임코드 |
| E5 | 완료 인물 서명이 식탁 중앙에서 공존 | 좌석 이름표·문양 연결선·착석 상태 | 식기 리듬 자막·빈 박자 표시 | 색 잔상 대신 소품 위치와 고정 연결선 |
| F2 | 아바타와 연구원 기록 음성의 겹침 | RESIDENT 이름·기능·문양 | 화자명·기록 출처·필수 사실 태그 | 음성 겹침 대신 아바타/원본 라벨 교대 |
| F3 | 두 장치의 실제 공기·가상 열기 대비 | 선택 장치에 사용인 색 자체를 쓰지 않음 | 확실·불확실 텍스트와 감각 자막 | 강제 카메라 없이 정적 대칭 배치 |

### 11.2 D5 안전 규칙

- 성공 표기 `SYNC COMPLETE`를 파열보다 먼저 텍스트로 보여 준다.
- 붉은 실패 X, 체력 손실, 재시도 버튼을 사용하지 않는다.
- 초회 HOLD 구간에도 일시정지·설정·대사 로그를 허용한다.
- 화면 흔들림은 0.4초 이하 한 번이며 멀미 감소에서는 제거한다.
- 광과민 보호에서는 점멸을 점진적인 텍스처 박리와 진단 라벨 변화로 대체한다.

### 11.3 E5·F2 결산 표현

| 등급 | 색상 사용 | 비색상 필수 표현 |
| --- | --- | --- |
| LOW | 채널이 각자 몸 주변에 고립 | 사용인은 서 있고 기능 라벨 중심 |
| MID | 완료 2~3채널만 중앙 접촉 | 완료 인물만 착석, 문양 연결 |
| HIGH | 네 채널 안정·한 채널 노이즈 | 네 자리 착석·회색 인덱스 한 자리 |
| ALL | 다섯 경계를 유지한 공존 | 다섯 이름·문양·음향을 순차 인증 |

- LOW를 탁하거나 나쁜 색으로, ALL을 밝거나 선한 색으로 표현하지 않는다.
- 미완료 인물도 고유 서명과 이름을 잃지 않는다.
- F2 필수 사실 태그는 `PROMISE`, `CONVERSION`, `FORCED ACTIVATION`, `OUTSIDE UNKNOWN`, `SUBJECT AUTHORITY` 텍스트로 제공한다.
- 질문을 생략하면 `F2_MANDATORY_RECAP`이 누락 태그만 읽는다.

### 11.4 F3·EDC 선택 중립 규격

| 항목 | 냉각 기상 | 안정화 잔류 | 허용 오차 |
| --- | ---: | ---: | ---: |
| 버튼 면적 | 100% 기준 | 100% 기준 | ±2% |
| 중심에서 거리 | 동일 | 동일 | ±4px |
| 기본 명도 대비 | 동일 단계 | 동일 단계 | WCAG 대비 단계 동일 |
| 확실한 정보 줄 수 | 2~4줄 | 2~4줄 | ±1줄 |
| 불확실한 정보 줄 수 | 2~4줄 | 2~4줄 | ±1줄 |
| 위험 아이콘 수 | 1 | 1 | 0 |
| 확인 횟수 | 1 | 1 | 0 |

중립 포커스 규칙:

1. F3 진입 포커스는 중앙 SUBJECT 수첩대다.
2. F3에서 마지막으로 조사한 장치를 EDC 기본 포커스로 사용하지 않는다.
3. `f0_provisional_intent`를 카메라·버튼·진동·문구 순서에 사용하지 않는다.
4. EDC 취소 뒤 중앙 SUBJECT 수첩대로 돌아오며 `FINAL_SLEEP_LOCK`은 유지한다.
5. 시스템 라벨은 `기상 절차 실행`, `안정화 루프 복원`을 사용한다. `용기`, `안전`, `도피`, `진짜 엔딩`을 사용하지 않는다.

## 12. 엔딩 상호작용 접근성

### 12.1 ED_ALL_CEREMONY 필기

| 항목 | 기본 | 대체 입력 |
| --- | --- | --- |
| 이름 인증 | 다섯 색·문양·음향 순차 재생 | 이름·기능·문양 텍스트 목록 |
| 서명 | 희미한 문장을 따라 한 번 긋기 | 버튼 1회 자동 추적 또는 길게 누르기 |
| 제한 | 오답·시간 제한 없음 | 입력 없이 4초 뒤 자동 추적 제안 |
| 중단 재개 | 첫 미완료 인증부터 | 같은 지점, 확정 분기 유지 |

필기 결과는 엔딩 허가나 성공을 판정하지 않는다. 운동 보조를 사용해도 장면·대사·메타 결과가 동일하다.

### 12.2 현실 연결 해제·호흡

- 첫 호흡은 자동 성공한다.
- 버튼 연타, 정확한 리듬, 마이크 입력, 호흡 유지 QTE를 사용하지 않는다.
- 기본 흔들림은 0.2초 이하이며 멀미 감소에서 제거한다.
- 검은 화면은 최대 4초이며 화면 읽기에는 `가상 연결 해제 중` 상태를 즉시 알린다.
- 소리 없이도 `압력 → 금속 맛 → 눈꺼풀 무게 → 첫 호흡`의 감각 순서를 텍스트로 제공한다.
- 숨소리 과민 설정에서는 실제 호흡음을 낮추고 파형·짧은 자막으로 대체한다.

### 12.3 현실 수첩

| 기능 | 규격 |
| --- | --- |
| 필수 열람 | 표지와 `첫 72시간` 요약만 |
| 긴 원문 | 별도 확대 패널, 필수 진행과 분리 |
| 화면 읽기 | 작성자→제목→요약→원문→뒤로 순서 |
| 색 제거 | 작성자 이름·기능 아이콘·문양 탭 병행 |
| 저시력 | 본문 확대 200%, 줄 간격 1.5, 가로 스크롤 금지 |
| 인지 보조 | `지금`, `첫날`, `나중` 세 시간 묶음 요약 |
| 동적 인계 | `SUBJECT_HANDOFF_PAGE`에 `프린터 출력물` 라벨 상시 표시 |

시뮬레이션 수첩과 현실 수첩은 서로 다른 아이콘·object_id·헤더를 사용한다. 표지 촉감의 유사성만으로 같은 물건처럼 안내하지 않는다.

### 12.4 현실 공간

- 신체 확인은 손·호흡 표시기·고정 벨트 중 고유 2종으로 완료한다.
- 포커스 순서는 캡슐 탈출에 필요한 물체를 따르되 특정 2종을 강제하지 않는다.
- 출구 패널의 전력·외기·수동 개폐는 색 대신 상태명과 아이콘으로 구분한다.
- 멀리 보이는 신호는 자동 확대·진동 유도·보상 아이콘을 사용하지 않는다.
- 선택 조사를 생략해도 마지막 화면과 메타 커밋에 도달한다.

### 12.5 S5 외형 표시

| 모드 | 기본 표현 | 접근성 보정 |
| --- | --- | --- |
| `layered` | 고딕 외피와 시설 골격 동시 표시 | 윤곽 복잡도·투명도 슬라이더 |
| `contextual` | 평소 고딕, 조사 시 시설 골격 | 전환 페이드 제거·즉시 라벨 |

- 두 모드는 언제든 바꿀 수 있으며 엔딩·관계·대사 결과가 같다.
- `contextual`에서도 `FILTER: DISPLAY ONLY`, `MODEL`, 실제 센서값 라벨을 숨기지 않는다.
- 광과민 보호에서는 레이어 전환에 점멸·스캔라인을 사용하지 않는다.
- 인지 보조에서는 `고딕 보기`, `구조 함께 보기`라는 보조 표시명을 병기한다.
- S5는 기존 중앙홀·식당에 적용되는 `world_phase`이며 새 공간처럼 길 찾기를 다시 학습시키지 않는다.

### 12.6 엔딩 저장·재개

| 중단 지점 | 재개 안내 |
| --- | --- |
| EDC 커밋 전 | 선택 미확정, F3 중앙 수첩대로 복귀 |
| EDC 커밋 뒤 | 확정 분기명과 다음 장면 표시 |
| ALL 의식 | 완료한 이름 인증 수와 자동 추적 설정 유지 |
| 현실 연결 해제 | `EDR_DISCONNECT` 시작에서 감각 순서 재생 |
| 에필로그 | 완료한 필수 오브젝트에 체크 표시, 다음 미완료 포커스 |
| 마지막 화면 뒤 | seen 메타를 먼저 저장하고 크레딧 진입 |

재개 UI는 이전 입력을 다시 요구하는 이유를 명시한다. 접근성 자동 입력 사용 여부는 저장하지만 엔딩 결과값으로 사용하지 않는다.

## 13. 아트·QA 체크리스트

- [ ] 회색조 캡처에서 다섯 서명이 구분된다.
- [ ] 각 서명에 고유 문양, 선, 음향, 라벨이 있다.
- [ ] 루카의 검정과 이리스의 흰색이 배경에서 사라지지 않는다.
- [ ] D5 글리치가 광과민 설정에서 점멸 없이 전달된다.
- [ ] D5가 오류·실패가 아니라 확정 성공 전환으로 읽힌다.
- [ ] 색 제거 모드로 P3B, C5, E3_5, F0-D를 완료할 수 있다.
- [ ] 색 제거·음량 0·글리치 0 조합에서 실행 짧은 반응 12종과 EDGAR_B2 세 하위 경로의 의미가 유지된다.
- [ ] IRIS_S2 네 계절과 MARA2_FU 세 선택에 기본 포커스·명도·색상 편향이 없다.
- [ ] 관계값 변경이 색상 가치 판단으로 읽히지 않는다.
- [ ] SUBJECT 표시가 어떤 사용인 서명과도 혼동되지 않는다.
- [ ] E5 LOW~ALL을 색 없이 좌석·문양·음향 자막으로 구분한다.
- [ ] F2 질문 생략 뒤 필수 사실 5종이 화면 읽기로 회수된다.
- [ ] F3·EDC 두 선택의 면적·거리·정보량·확인 횟수가 허용 오차 안이다.
- [ ] F0-E 의향과 마지막 조사 순서가 EDC 포커스를 바꾸지 않는다.
- [ ] ALL 의식 자동 필기가 수동 필기와 같은 장면 결과를 낸다.
- [ ] 현실 첫 호흡이 모든 입력 설정에서 자동 성공한다.
- [ ] 현실 수첩의 표지·72시간 요약이 화면 읽기와 200% 확대에서 잘리지 않는다.
- [ ] 시뮬레이션 수첩과 현실 수첩이 색 없이도 별도 매체로 구분된다.
- [ ] S5 layered·contextual 모드가 점멸 없이 전환되고 결과값을 바꾸지 않는다.
- [ ] 엔딩 중단 재개가 완료 상호작용과 접근성 설정을 보존한다.
