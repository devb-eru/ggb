# GGB v0.4 전체 이벤트 리스트·상태표

## 1. 문서 목적

본 문서는 GGB v0.4에서 사용하는 실제 이벤트, 시스템 전환, 조건형 하위 이벤트, 상태 변수와 흐름 보조 노드를 등록하는 단일 레지스트리다.

담당 범위:

- 이벤트 ID의 의미와 데이터용 표기.
- 이벤트의 필수성, 위치, 시간, 선행 조건과 결과.
- 실패·반복·재진입 정책.
- 영구·루프·파열·파생 상태.
- 사용인 관계·기억·개입 상태.
- J4·E5·F2·엔딩 변형.
- `03` 흐름도의 보조 노드와 실제 콘텐츠 이벤트 구분.

본 문서는 이벤트 내용을 장면 단위로 서술하지 않는다. 상세 내용은 `06~15`, 구현 자료형은 `17`이 담당한다.

## 3. 이벤트 개체 분류

### 3.1 콘텐츠 이벤트

플레이어가 직접 보고, 조사하고, 조작하거나 선택하는 장면이다.

예시:

- `P3B` 초상화 이름표 정리.
- `B3-B` 역할·위상 입력.
- `E3_2` 이리스 센서 교정.
- `F2` 연구원 대면.

등록 요건:

- 고유 ID.
- 선행 조건.
- 위치·시간.
- 성공·실패 또는 종료 조건.
- 저장 결과.
- 재진입 정책.

### 3.2 시스템 이벤트

세계 상태를 바꾸지만 독립적인 플레이 장면이 아닌 처리다.

예시:

- `NORMAL_RESET`.
- `SYS_COMMIT`.
- `ROUTE`.
- `BROKEN_RESET`.

시스템 이벤트도 디버그와 저장 복구를 위해 ID를 가진다.

### 3.3 오버레이 이벤트

다른 콘텐츠 이벤트 위에 겹쳐 발생하는 학습·연출이다.

예시:

- `CLR-00` 색·문양 학습.
- `CLR-03` D5 색상 서명 분리.
- 사용인별 엔딩 반응.

오버레이는 부모 이벤트 완료 없이 단독 완료할 수 없다.

### 3.4 조건형 하위 이벤트

메인 이벤트 사이에 조건이 맞을 때 재생되는 짧은 관계 장면이다.

예시:

- `EDGAR_S1`.
- `MARA2_S2`.
- `MARA2_FU`.

메인 진행을 차단하지 않으며 관계 변화는 최초 완료 때 한 번만 적용한다.

### 3.5 변형

동일한 이벤트 본체가 상태에 따라 다른 장면을 출력한다.

```text
event_id: E5
variant_id: LOW | MID | HIGH | ALL
```

변형을 별도 메인 이벤트로 중복 등록하지 않는다.

적용:

- `J4`: BASE, EXPANDED, FULL.
- `E5`: LOW, MID, HIGH, ALL.
- `F2`: LOW, MID, HIGH, ALL.
- `ED_REALITY`: LOW, MID, HIGH, ALL.
- `ED_STAY`: LOW, MID, HIGH, ALL.

### 3.6 흐름 보조 노드

`CHK_*`, `REL_*`, `SYS_*`, `ENTRY_*`, `MERGE_*`는 `03`의 조건·집계·합류를 표현한다.

- `CHK_*`: 상태를 읽고 분기.
- `REL_*`: 기존 대사·행동 조각을 출력.
- `SYS_*`: 상태 커밋·압축·동기화.
- `ENTRY_*`: ROUTE 진입점.
- `MERGE_*`: 분기 합류점.

보조 노드는 별도 장면 리소스가 없으면 콘텐츠 이벤트 완료율에 포함하지 않는다.

### 3.7 비실행 기획 그룹

여러 실제 이벤트를 문서에서 한 묶음으로 부를 때 쓰는 별칭이다. 저장·대기열·완료 판정에는 그룹명을 쓰지 않는다.

| 그룹명 | 실제 이벤트 ID | 용도 | 금지 |
| --- | --- | --- | --- |
| `EDGAR_B2` | `B2_EDGAR_ENTRY`, `B2_HIDE`, `B2_CAUGHT` | 기록 내실 압박 반응 묶음 | `EDGAR_B2_seen`, 단일 이벤트 리소스, 단일 완료 플래그 생성 |

`EDGAR_B2`의 관계·잔류 기억·반복 축약은 실제로 재생된 하위 이벤트 ID를 기준으로 기록한다.

## 4. ID 규칙

### 4.1 기획 ID와 데이터 ID

| 기획 표기 | 데이터 표기 |
| --- | --- |
| `B3-A` | `B3_A` |
| `C2-1` | `C2_1` |
| `F0-D` | `F0_D` |
| `CLR-04` | `CLR_04` |

Markdown 본문과 플레이어용 디버그 표시는 기획 ID를 사용한다. 파일명, 리소스명, 변수명은 데이터 ID를 사용한다.

### 4.2 접두어

| 접두어 | 의미 |
| --- | --- |
| `P` | 프롤로그 |
| `A~D` | 정상 리셋 구간 |
| `J` | 일지 복원 |
| `E` | 파열 이후 관계·결산 |
| `F` | 코어·최종 진실 |
| `ED` | 엔딩 |
| `CLR` | 색상 서명 단계 |
| `REC` | 연구원 기록 |
| `MEM` | 잔류 기억 |
| `SYS` | 시스템 처리 |
| `CHK` | 조건 판정 |
| `REL` | 관계 반응 |
| `ENTRY` | 구간 진입 |
| `MERGE` | 분기 합류 |

### 4.3 와일드카드

`MARA2_ED_*` 같은 표기는 문서 요약에만 사용한다. 실제 데이터에는 구체 ID를 등록한다.

```text
MARA2_ED_REALITY
MARA2_ED_STAY
```

## 6. 이벤트 생명 주기

```text
LOCKED
→ AVAILABLE
→ ACTIVE
→ COMPLETED
```

실패가 있는 경우:

```text
ACTIVE
→ FAILED_FOR_LOOP
→ NORMAL_RESET
→ AVAILABLE
```

LOCAL RETRY:

```text
ACTIVE
→ RETRY_READY
→ ACTIVE
```

상태:

| 상태 | 의미 |
| --- | --- |
| `locked` | 선행 조건 미충족 |
| `available` | 시작 가능 |
| `active` | 현재 진행 중 |
| `choice_pending` | 기술 퍼즐 완료, 관계 선택 미확정 |
| `completed` | 완료 결과 커밋 |
| `failed_for_loop` | 당일 재시도 불가 |
| `retry_ready` | 현장 재시도 가능 |
| `superseded` | 이후 상태가 대체 |

### 7.1 P4 부모·하위 장면 커밋 규칙

`P4_IRIS_GREETING`은 별도 메인 이벤트가 아니라 P4 종료부에 자동 재생되는 필수 하위 장면이다.

```yaml
event_id: P4_IRIS_GREETING
event_type: sub_event
parent_event_id: P4
location_id: M1_KITCHEN
required: true
prerequisites:
  all:
    - P4_complete
outputs:
  set_flags:
    - iris_greeting_seen
  add_servants:
    - IRIS
  unlock_optional_objectives:
    - P5
```

커밋 규칙:

- 차 준비가 끝나면 P4의 조작 구간을 완료하고 `P4_IRIS_GREETING`을 즉시 호출한다.
- 하위 장면에는 거절·이탈 선택지를 두지 않는다.
- P4 완료 직후 저장이 중단되어 `iris_greeting_seen=false`라면 로드 시 하위 장면부터 복구한다.
- P5는 선택이지만 P6은 `iris_greeting_seen=true`를 요구한다.
- 하위 장면은 콘텐츠 완료율에서 P4와 별도 메인 이벤트로 중복 집계하지 않는다.
- `introduced_servants`에는 하위 장면 완료 시 IRIS를 추가한다.

### 21.3 위장 필터 상태 전환

| 현재 상태 | 사건·가드 | 다음 상태 | 함께 확정하는 상태 | 수면 라우팅 |
| --- | --- | --- | --- | --- |
| `ACTIVE` | D5 이전 `NORMAL_SLEEP` | `ACTIVE` | 없음 | `NORMAL_SLEEP` |
| `ACTIVE` | D5 완료 | `DISABLED` | `D5_complete=true` | 다음 수면은 `FRACTURE_SLEEP` |
| `DISABLED` | D6·`FRACTURE_SLEEP` | `DISABLED` | 정상 RESET 금지 | `BROKEN_RESET` 실행 |
| `DISABLED` | `BROKEN_RESET_ONCE` 완료 | `BROKEN` | `broken_reset_triggered=true`, `fracture_sleep_complete=true`, S3 1회 생성 | 이후 `POST_BROKEN_REST` |
| `BROKEN` | `POST_BROKEN_REST` | `BROKEN` | 시간·피로만 변경 | `POST_BROKEN_REST` 반복 |

`DISABLED`는 D5가 위장 출력을 끈 뒤 아직 복구 시도를 확정하지 않은 상태이고, `BROKEN`은 `BROKEN_RESET_ONCE`가 복구 불능과 S3 생성을 원자적으로 확정한 상태다. `camouflage_filter_enabled`와 `camouflage_filter_disabled`는 더 이상 저장하거나 읽지 않는다.

불변식:

```text
camouflage_filter_state == ACTIVE   ⇒ !D5_complete && !broken_reset_triggered
camouflage_filter_state == DISABLED ⇒  D5_complete && !broken_reset_triggered
camouflage_filter_state == BROKEN   ⇔  broken_reset_triggered
```

## 30. 이벤트 우선순위

한 공간 진입에서 여러 이벤트가 가능하면 다음 순서를 사용한다.

```text
1. 필수 메인 전환
2. 현재 활성 퍼즐 결과
3. 핵심 관계 이벤트
4. 처음 보는 짧은 관계 이벤트
5. 잔류 기억 반응
6. 색상 이상
7. 반복 요약
8. 기본 오브젝트 반응
```

규칙:

- 자동 대화는 공간 진입당 최대 1개.
- 미확인 반응은 `pending_reactions`에 남긴다.
- 플레이어는 선택형 반응을 나중에 볼 수 있다.
- J4 확정 시 미완료 E구간 핵심 관계 이벤트는 `superseded` 처리하되 완료로 계산하지 않는다.

## 31. 반복·재진입 정책

| 정책 | 사용 |
| --- | --- |
| once | 일지, 핵심 관계, 첫 짧은 반응 |
| per_loop | 수면, 일과, 제조, 개입 예산 |
| until_success | 필수 퍼즐·필수 조사 |
| ambient | 공통 오브젝트 반응 |
| repeatable | E_HUB, 후반 휴식 |

관계 이벤트 재진입:

- 완료 전 중도 이탈: 이어서 진행 또는 처음부터 재시작을 이벤트별 지정.
- 완료 후: 결과 확인과 후속 대사만 제공.
- 관계 변화 재적용 금지.

## 32. 리셋 범위

| 데이터 | NORMAL_RESET | BROKEN_RESET의 `BROKEN_RESET_ONCE` | POST_BROKEN_REST |

| 물리 월드 | S0 기준 초기화 | S3 기준 생성 | 유지 |

| 관계·기억 | 유지 | 유지 | 유지 |

## 34. 참조 무결성 규칙

1. `03`의 실제 콘텐츠·시스템 이벤트는 본 레지스트리에 존재한다.
2. `CHK/REL/SYS/ENTRY/MERGE`는 보조 노드 표에 존재한다.
3. 모든 콘텐츠 이벤트는 상세 문서를 참조한다.
4. 모든 플래그는 쓰는 이벤트와 읽는 이벤트가 있다.
5. 선택 이벤트는 F0·EDC의 선행 조건이 될 수 없다.
6. 변형 ID는 부모 이벤트 없이 단독 실행할 수 없다.
7. 색상 오버레이는 부모 이벤트 결과에서 계산한다.
8. 와일드카드 ID는 구현 전 구체 ID로 변환한다.
9. 완료된 E3_1~E3_5의 `event_history.outcome_id`는 §13.3 허용값 중 하나다.
10. `E3_X_complete`, REC_OWNER, 사용인 핵심 완료·기록 획득, 관계 적용 이력은 모두 일치한다.
11. E3_4M은 `E3_4_complete`, REC_EDGAR, 관계 수치, 핵심 관계 완료 수를 쓰지 않는다.
