# GGB v0.4 이벤트 상세 08: 파열·전환·결산

## 1. 전환 목적

D5는 “퍼즐 실패”가 아니라 고딕 위장 필터를 해제하는 예정된 전환이다. D5 전에는 수면이 정상 세계를 복구하지만, D5 이후에는 같은 복구 명령이 손상된 기반 상태를 불러와 다른 아침을 만든다.

## 2. D5 세계의 파열

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 지하창고 태엽 심장실, D4 성공 직후 |
| 선행 조건 | 태엽 심장 XIII 기동 |
| 목표 | 위장 필터 해제와 장르 전환 체험 |
| 상호작용 | 자동 전환 중 시선 이동, 사용자 입력은 관찰만 허용 |
| 성공·실패 | 확정 전환. 실패 판정 없음 |
| 힌트 | D4의 `CAMOUFLAGE FILTER OFF` |
| 관계 변화 | 없음 |
| 색상 반응 | 다섯 색이 장식에서 분리되어 배선과 인격 잔상으로 노출 |
| 감각 연출 | 벽지가 프레임 단위로 금속 격자에 붙었다 떨어지고, 향초 냄새가 오존 냄새로 치환 |
| 구현 메모 | 90~150초. 광과민 설정에서는 점멸 대신 점진적 마스크 박리 사용 |

연출 순서:

1. 태엽 심장은 정상 작동음을 낸다.
2. 저택 표면의 고딕 텍스처가 깊이값을 잃는다.
3. 사용인의 그림자에서 고유 색 서명이 반 박자 늦게 분리된다.
4. 창밖 밤하늘이 정지된 배경 레이어로 드러난다.
5. 침실 방향에서 수면 권유 방송이 자동 재생된다.

## 4. FRACTURE_SLEEP와 BROKEN_RESET

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 로딩·상태 전환 |
| 선행 조건 | D6 |
| 목표 | 정상 리셋 규칙을 파열 상태 규칙으로 교체 |
| 상호작용 | 없음 |
| 성공·실패 | 항상 성공 |
| 힌트 | 리셋 체크리스트에서 `WORLD MASK`만 실패 표시 |
| 관계 변화 | 없음 |
| 색상 반응 | 다섯 채널은 `PERSIST`, 고딕 필터는 `FAILED` |
| 감각 연출 | 종소리가 열두 번을 끝내지 못하고 냉각 장치 팬음으로 이어짐 |
| 구현 메모 | `broken_reset_triggered = true`; 이후 일반 RESET 노드 사용 금지 |

상태 차이:

| 항목 | 정상 RESET | BROKEN_RESET |
| --- | --- | --- |
| 물리 배치 | S0 기준 복구 | S3 손상 기준 복구 |
| 고딕 필터 | 재적용 | 복구 실패 |
| 수첩·일지 | 유지 | 유지 |
| 사용인 기억·관계 | 유지 | 유지 |
| 색 서명 | 장식으로 은폐 | 데이터 잔상으로 노출 |
| 당일 기믹 | 초기화 | E구간 로컬 장치만 초기화 |

| 구현 메모 | 4~7분; 과도한 점프 스케어 금지 |

## 6. E2 사용인 대면

E2는 [12번 문서](12_이벤트상세_07_사용인핵심관계.md)의 `E2_INTRO`로 구현한다. 여기서는 전환 기능만 정의한다.

- 사용인들은 파열 이전 기억을 숨기지 못한다.
- 에드가는 강제 복구가 불가능함을 인정한다.
- 마라 2는 고딕 외형을 “너무 오래 쓴 표지”라고 부른다.
- 각자 장치를 안정화하면 연구원 기록을 읽을 수 있다는 선택 구조가 열린다.
- 바로 E5 결산으로 가도 메인 진행은 가능하다.

## 7. J4 결산 일지

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 서재, E_HUB에서 조사 종료 후 |
| 선행 조건 | E2_INTRO, E3_4 또는 E3_4M |
| 목표 | 획득한 연구원 기록 수에 따라 아버지의 배신과 미완성 계획을 정리 |
| 상호작용 | 기록 페이지 순서 배열, 모순 문장 확인 |
| 성공·실패 | 기록 수와 무관하게 완료. 배열 오답은 로컬 재시도 |
| 힌트 | 날짜, 승인 번호, 고유 문양 |
| 관계 변화 | 없음 |
| 색상 반응 | 기록 소유자의 서명만 활성화; 미획득은 회색 인덱스 |
| 감각 연출 | 종이 넘김 소리 뒤로 실제 서버 헤드 이동음 |
| 구현 메모 | `J4_BASE`, `J4_EXPANDED`, `J4_FULL` 중 하나 |

| 기록 수 | 내용 |
| --- | --- |
| 0~1 | 아버지가 약속을 어겼고 연구원 인격이 시스템에 남았다는 사실 |
| 2~4 | 각자의 동의 범위, 주인공 보존 계획, 해제 기준 부재 |
| 5 | 마라 2의 분산 체크섬을 포함한 전환 전모와 연구원 해방 계획 초안 |

J4는 과거 사건의 기록 결산이다. F1/J5는 현재 주인공의 선택 권한을 복원하는 기록이므로 역할을 분리한다.

## 8. E5 마지막으로 정상인 저녁

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 중앙홀·식당, E구간 마지막 저녁 |
| 선행 조건 | J4 완료 |
| 목표 | 관계 완료 수에 따른 사용인 집단 결산 |
| 상호작용 | 좌석 선택, 한 번의 공동 질문, E6 진입 확인 |
| 성공·실패 | 확인 전 자유 이동 가능. 확인 후 F구간으로 진행 |
| 힌트 | 미완료 이벤트 수와 예상 시간 표시 |
| 관계 변화 | 없음 |
| 색상 반응 | LOW/MID/HIGH/전원 완료에 따라 채널 안정도가 다름 |
| 감각 연출 | 식기는 고딕 외형과 금속 골격 사이를 천천히 오감 |
| 구현 메모 | 5~12분; 놓친 관계 이벤트 경고는 서사 밖 UI로 명확히 제공 |

### LOW: 0~1명

- 사용인은 각자 자리에서 시스템 보고만 한다.
- 대화 사이의 공백이 길고, 색 잔상은 서로 겹치지 않는다.
- 에드가가 E6을 업무적으로 개방한다.

### MID: 2~3명

- 완료한 사용인이 미완료 사용인의 문장을 대신 잇는다.
- 일부 색이 식탁 중앙에서 잠시 공존한다.
- “보내야 하는가”에 대한 의견 차이가 노출된다.

### HIGH: 4~5명

- 각자가 자신의 감금 책임과 주인공에 대한 애착을 직접 인정한다.
- 다섯 번째가 미완료면 해당 자리는 불안정한 회색 인덱스로 남는다.
- E6 개방은 합의된 선택 반환으로 표현된다.

### 전원 완료

- `all_servants_complete = true`.
- 다섯 고유 음향이 순차 재생된 뒤 하나의 화음으로 끝난다.
- 색은 섞여 단일색이 되지 않고 각자 구분된 채 한 프레임에 공존한다.
- 마라 2가 모두의 이름을 불러 확인하고, 마지막에 주인공에게 자신의 임시 호칭을 되묻는다.

## 9. F2 연구원들과의 대면

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 코어실 전면, F1/J5 후 |
| 선행 조건 | `subject_authority_restored` 이전, J5 완료 |
| 목표 | 연구원들이 감금 책임을 인정하고 최종 선택권을 반환 |
| 상호작용 | 책임, 현실 위험, 시뮬레이션 유지 가능성 질문 |
| 성공·실패 | 모든 질문은 선택. 대화 종료 시 F3 |
| 힌트 | 없음 |
| 관계 변화 | 없음 |
| 색상 반응 | 완료한 사용인은 안정, 미완료는 노이즈가 섞인 서명 |
| 감각 연출 | 목소리가 사용인 외형과 연구원 기록 음성 사이를 오감 |
| 구현 메모 | 완료 수별 길이 4~10분; 미완료여도 핵심 사실 누락 금지 |

필수 공개:

1. 아버지는 연구원을 미래 육체로 옮기겠다고 약속했으나 통 속의 뇌와 인격 데이터로 전환했다.
2. 주인공은 외부 환경이 회복될 때까지 냉각 보존될 예정이었다.
3. 아버지 사망 뒤 사용인들은 외로움, 원망, 보호 욕구로 주인공을 시뮬레이션에 강제 접속했다.
4. 현재 외부 생존 가능성은 불명확하며 안전을 보장할 수 없다.
5. 최종 선택은 주인공에게 돌아가야 한다.

### MARA2_F2

- E3_5 미완료: 보라 익명 인덱스가 기록 조작 로그를 읽고, 마라 2는 자신의 손상 원인을 확신하지 못한다.
- 병합 완료: 마라 2는 동료 기억을 지키려 스스로 영역을 내줬다고 감정적으로 인정한다.
- 분리 보존 완료: 원본과 감정 주석이 번갈아 말하되 서로를 같은 자신으로 인정한다.
