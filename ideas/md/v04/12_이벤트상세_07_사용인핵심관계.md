# GGB v0.4 이벤트 상세 07: 사용인 핵심 관계

## 1. 역할

핵심 관계 이벤트는 D5 이후 사용인의 연구원 시절과 현재 의무를 연결한다. 모두 선택형이며 메인 진행, F0, 엔딩 선택지를 차단하지 않는다.

권장 흐름:

```text
E2_INTRO
→ E_HUB
→ 원하는 E3 이벤트
→ 기록 획득
→ E_HUB 복귀
→ 계속 조사 또는 결산
```

완료 수 목표:

| 완료 | 결산 |
| --- | --- |
| 0~1 | LOW |
| 2~3 | MID |
| 4~5 | HIGH |
| 5 | HIGH + `all_servants_complete` |

## 2. E2_INTRO 파열 후 합의

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 중앙홀, E1 아침 직후 |
| 선행 조건 | `broken_reset_triggered` |
| 목표 | 다섯 사용인의 장치 상태와 선택 조사 구조 제시 |
| 상호작용 | 저택 상태, 주인공 상태, 아버지 기록 중 질문 순서 선택 |
| 성공·실패 | 대화 종료 시 E_HUB 개방. 실패 없음 |
| 힌트 | 다섯 서명과 대응 공간을 지도에 표시 |
| 관계 변화 | 없음 |
| 색상 반응 | 다섯 색이 각자의 몸과 반 박자 어긋나 노출 |
| 감각 연출 | 같은 홀에 서 있으나 목소리 잔향의 방향이 다름 |
| 구현 메모 | `mara2_archive_index_known`가 없으면 보라 익명 인덱스 자동 제공 |

에드가는 장치 안정화를 명분으로 각자에게 흩어질 것을 제안한다. 실제로는 주인공과 연구원이 일대일로 대화할 기회를 주는 절충이다.

## 3. E_HUB 관계 선택 허브

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 중앙홀, E구간 자유 시간 |
| 선행 조건 | E2_INTRO 완료 |
| 목표 | 다음 사용인 이벤트 또는 결산 선택 |
| 상호작용 | 지도 핀 선택, 사용인에게 이동, “조사를 마친다” 선택 |
| 성공·실패 | 어느 시점에도 조사를 끝낼 수 있음 |
| 힌트 | 미완료 사용인, 예상 시간, 관련 공간만 표시하고 보상은 숨김 |
| 관계 변화 | 없음 |
| 색상 반응 | 완료 핀은 색+문양이 안정된 이중선으로 변함 |
| 감각 연출 | 선택한 핀의 고유 음향이 멀리서 한 번 응답 |
| 구현 메모 | 완료 이벤트 재진입 금지, 후속 대화만 허용 |

| 성공·실패 | 잘못 벗긴 피복은 로컬 복구. 완료를 포기해도 허브 복귀 |

| 색상 반응 | 주황 대각선이 삭제된 문장을 쓸어내지 않고 밑줄로 남김 |
| 감각 연출 | 탄 피복 냄새와 마른 솔 소리가 겹침 |
| 구현 메모 | 9~14분; `REC_MARA1`, `E3_1_complete` |

연구원 기록 핵심:

- 마라 1은 아버지의 명령으로 실험 실패 기록을 지웠다.
- 시뮬레이션 전환에 동의했지만 “육체 포기” 조항은 보지 못했다.
- 현재의 청소 강박은 삭제 업무가 의례로 변한 결과다.

## 5. E3_2 이리스: 계절 센서 교정

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 온실 계절 제어실, E구간 |
| 선행 조건 | E2_INTRO, E3_2 미완료 |
| 목표 | 센서 입력과 연출 날씨를 분리해 실제 외부값 일부 복구 |
| 상호작용 | 온도·습도·광량 센서 교차 검증, IRIS_S2의 계절 기억 선택 재확인 |
| 성공·실패 | 잘못 교정하면 가짜 계절 연출만 바뀌며 즉시 재시도 |
| 힌트 | 꽃잎 문양은 연출, 유리 진동은 외부 센서 |
| 관계 변화 | 불완전한 기억 인정 `bond +2`; 이상적 계절 강요 `alert +1` |
| 색상 반응 | 흰 후광과 연노랑 꽃잎이 분리되었다가 한 프레임에 공존 |
| 감각 연출 | 따뜻한 빛 아래 차가운 공기, 젖지 않는 흙냄새 |
| 구현 메모 | 10~15분; `REC_IRIS`, `E3_2_complete` |

기록 핵심:

- 이리스는 외부 생태 복구 예측 담당이었다.
- 지구가 안전해졌다는 보고는 없고, 단지 “언젠가 가능”이라는 모델만 남았다.
- 주인공이 나갈 수 있다는 희망과 지금은 위험하다는 공포를 동시에 가진다.

## 6. E3_3 루카: 생명 유지 장치

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | 주방 하부 생명 유지실, E구간 |
| 선행 조건 | E2_INTRO, E3_3 미완료 |
| 목표 | 유체 압력을 안정화하고 주인공 냉각 장치 연결을 확인 |
| 상호작용 | 검정 주관과 연두 보조관의 맥박 위상 정렬, 경고 로그 열람 |
| 성공·실패 | 위상 오차 시 안전 밸브가 열리고 로컬 재시도 |

| 관계 변화 | 진실 공개 요구 `bond +2, alert +1`; 현재 안정 우선 `bond +1, alert -1` |

| 감각 연출 | 냉각액 냄새, 혀끝의 금속 맛, 손목과 배관의 동시 박동 |

기록 핵심:

- 루카는 주인공의 냉각 보존 계획에 참여했다.
- 목적은 멸망기의 독성 대기와 방사선 수치가 낮아질 때까지 생존시키는 것이었다.
- 아버지 사후 해제 기준이 불명확해 사용인들이 자의적으로 시뮬레이션을 연장했다.

## 7. E3_4 에드가: 대시계 보안 코어

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | `M1_GREAT_CLOCK` → `H0_CLOCK_MACHINE`, E구간 유연 |
| 선행 조건 | E2_INTRO, E3_4 미완료 |
| 목표 | 파열된 코어 권한을 재배치하고 에드가의 감금 책임 확인 |
| 상호작용 | 남색 잠금선 4개를 `보호/감시/기억/선택` 권한에 배치 |
| 성공·실패 | 모순 배치 시 로그가 충돌하며 로컬 재시도 |
| 힌트 | 보호와 선택은 같은 주체에게 귀속될 수 없다는 보안 문구 |
| 관계 변화 | 에드가에게 책임 유지 요구 `bond +1, alert -1`; 주인공에게 선택 반환 `bond +2, alert +1` |
| 색상 반응 | 수직 잠금선 하나가 주인공 수첩의 연필선으로 바뀜 |
| 감각 연출 | 낮은 시계음이 심장 박동 속도로 느려짐 |
| 구현 메모 | 12~18분; `REC_EDGAR`, `E3_4_complete`, E6 대사 변형 |

기록 핵심:

- 에드가는 연구팀의 운영·윤리 책임자였다.
- 아버지를 믿고 동료들의 전환 동의를 취합했으며, 결과를 막지 못했다.
- 아버지 사후 주인공을 깨우지 않은 최종 결정은 에드가가 내렸다.

## 8. E3_4M 에드가 최소 진행

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | `M1_GREAT_CLOCK` → `H0_CLOCK_MACHINE`, J4 뒤·E5 전 |
| 선행 조건 | J4 완료, E3_4 미완료, E5 진입 전 |
| 목표 | E6 개방에 필요한 운영 권한만 복구 |
| 상호작용 | 남색 코어핀 한 개를 원위치 |
| 성공·실패 | 실패 없음, 1~2분 |
| 힌트 | 에드가 직접 안내 |
| 관계 변화 | 없음; `core_event_complete`와 기록을 지급하지 않음 |
| 색상 반응 | 잠금선 하나만 복구되고 나머지는 흔들림 |
| 감각 연출 | 짧고 무감정한 잠금 해제음 |
| 구현 메모 | `edgar_minimum_access`; J4 과거 결산 뒤 CHK_EDGAR_CORE에서만 호출. E3_4는 E5 진입 전까지 선택 가능하나 E5 진입 후 종료 |

`E3_4M`은 관계 이벤트의 축소판이 아니라 `LOCK_CORE`의 최소 권한 복구다. E6은 `J4 완료 && (E3_4_complete || edgar_minimum_access) && e5_locked_in`을 요구하며, bond·alert·다른 사용인의 완료 수로는 막히지 않는다.

## 9. E3_5 마라 2: 원본과 손상 복사본

| 항목 | 내용 |
| --- | --- |
| 위치·시간 | `M1_COLOR_ROOM_ENTRY` 전환 → `H0_COLOR_SEPARATION` → `H0_PERSONALITY_ARCHIVE`, E구간 유연 |
| 선행 조건 | E2_INTRO, E3_5 미완료 |
| 목표 | 다섯 출처를 분리하고 마라 2 원본 인격의 결손 원인 확인 |
| 상호작용 | 색·문양·음향 필터, 초상화 3장 중첩, 체크섬 비교, 보존 방식 선택 |
| 성공·실패 | 오조합은 로컬 재시도. 중도 이탈 가능 |
| 힌트 | 보라 이중 윤곽, 빠른 3음, 누락된 셋째 구간, 다른 네 기록의 동일 체크섬 |
| 관계 변화 | 병합 `bond +2, alert +1`; 분리 보존 `bond +1, alert -1` |
| 색상 반응 | 보라 바깥선에 흩어진 네 색 조각이 원래 소유자에게 돌아감 |
| 감각 연출 | 겹친 목소리가 하나씩 떨어져 나가고 마지막에 마라 2 목소리만 가까이 남음 |
| 구현 메모 | 10~16분; `REC_MARA2`, `E3_5_complete`; 색 제거 모드에서는 문양·파형·라벨 사용 |

진입 규칙:

- 고딕 외부문과 C5 이후의 렌즈 조사는 `M1_COLOR_ROOM_ENTRY`에 남는다.
- 실제 분리 조작은 BROKEN_RESET 뒤 `H0_COLOR_SEPARATION`에서 시작한다.
- 체크섬 문을 통과한 뒤에만 `H0_PERSONALITY_ARCHIVE`로 이동한다.
- E3_5를 시작하지 않거나 중도 이탈해도 E2_INTRO의 `익명 보라 인덱스`가 F0-D의 최소 출처 데이터를 제공한다. 따라서 E3_5는 LOCK_CORE·F0·엔딩의 게이트가 아니다.

단계:

1. 단체 초상화 3장에서 다섯 서명을 분리한다.
2. 보라 채널의 시간순 조각을 연결한다.
3. 체크섬 12칸 중 비어 있는 3칸을 확인한다.
4. 네 사용인 아카이브에 같은 조각이 분산 저장되어 있음을 찾는다.
5. 마라 2가 동료 기억 보존을 위해 자기 감정 주석 영역을 양도한 사실을 확인한다.
6. 원본에 감정 주석을 병합하거나 별도 사본으로 보존한다.

장면 핵심:

마라 2는 작업 내내 자신이 예상한 결과라며 허세를 부리지만, 복원된 이름 필드가 잠깐 비면 말을 멈춘다. 주인공이 이름을 불러 준 뒤에야 다시 장난스러운 말투로 돌아온다.

## 11. 관계 값과 메인 개입

- `bond`는 대사 공개량, 도움의 자발성, 엔딩 반응에만 영향.
- `alert`는 사용인의 경계 대사와 당일 개입 예산에 영향.
- 어떤 값도 퍼즐 정답을 바꾸지 않는다.
- 높은 `bond`는 이미 검증한 준비 동작을 대신 수행할 수 있다.
- 높은 `alert`는 질문·확인 절차를 늘리지만 반드시 우회 가능한 일정 공백을 제공한다.
- E3 완료 후 사용인은 주인공의 최종 선택을 방해하지 않는다.
