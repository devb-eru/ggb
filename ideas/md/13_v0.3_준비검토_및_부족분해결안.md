# GGB v0.3 준비 검토 및 부족분 해결안

## 1. 문서 목적

이 문서는 이전 문서에서 제시한 문제와 권장안을 모두 채택한 뒤, 기획서 v0.3 작성 전에 추가로 필요한 것이 남아 있는지 검토한다.

결론부터 말하면, v0.3 작성을 막는 구조적 충돌은 해소되었다. 다만 v0.3 본문에 들어갈 세부 자료가 아직 필요하므로, 아래 최종 권장 작업을 순차적으로 작성한다.

## 2. 채택 완료한 권장안

| 항목 | 채택 내용 |
| --- | --- |
| 연구원 기록 3개 조건 | 메인 필수 게이트에서 제거하고 권장 진행 조건으로만 유지 |
| 사용인 관계 시스템 | 메인 진행 차단이 아니라 정서, 힌트, 엔딩 연출 변화로 사용 |
| 에드가 이벤트 | 전체 관계 이벤트는 선택형이지만 코어 접근 전 최소 대면은 반필수 |
| D5 이후 잠들기 | 정상 리셋이 아니라 `BROKEN_RESET`을 여는 핵심 트리거 |
| 관계 수치 표현 | 숫자 UI 없이 유대도/경계도를 감각적 피드백으로 표현 |
| 사용인 기억 유지 | 잔류 기억은 유지하되 표층 역할과 물리 상태는 리셋 |
| 사용인 개입 | 하루 1회 강한 개입을 기준으로 제한 |
| 엔딩 구조 | `ED_REALITY`, `ED_STAY` 두 선택 유지, 내부 연출만 변화 |

## 4. 스토리 및 흐름도 부족분 채택 결과

### [S1] 챕터 구분과 이벤트 코드가 아직 완전히 맞물리지 않는다

#### 문제

현재 흐름도는 `P/A/B/C/D/E/F` 단계로 잘 정리되어 있지만, 실제 기획서 본문에서는 프롤로그와 1~4챕터 구조가 필요하다.

#### 권장안

아래처럼 매핑한다.

| 기획서 챕터 | 흐름 코드 | 역할 |
| --- | --- | --- |
| 프롤로그 | P | 저택 일상, 첫 취침, 첫 리셋 전 감정 각인 |
| 1챕터 | A~B | 리셋 검증, 수첩, 시간표, 열세 번째 종 |
| 2챕터 | C~D | 거울, 지하창고, 태엽 심장, D5 세계의 파열 |
| 3챕터 | E | BROKEN_RESET 이후 사용인 관계 이벤트 |
| 4챕터 | F~ED | 코어 접근, 아버지 기록, 연구원 대면, 최종 선택 |

#### 채택 결과

채택 완료. v0.3에서는 프롤로그와 1~4챕터 구조를 공식 구조로 사용하고, `P/A/B/C/D/E/F` 코드는 내부 흐름 코드로 병행 표기한다.

### [S2] 관계 이벤트의 플레이타임이 커질 위험이 있다

#### 문제

사용인 4명 모두에게 관계 이벤트를 넣으면 플레이타임이 급격히 늘어날 수 있다.

#### 권장안

- D5 이전 이벤트는 30~60초짜리 짧은 반응으로 제한한다.
- D5 이후 핵심 관계 이벤트는 사용인당 3~5분으로 제한한다.
- 전체 관계 이벤트를 모두 보아도 추가 플레이타임이 20분 안쪽에 머물도록 한다.

#### 채택 결과

채택 완료. v0.3에서는 사용인별 관계 이벤트를 `D5 이전 짧은 반응 2개`, `D5 이후 핵심 이벤트 1개`, `후속 대사 1개`, `최종부 반영 1회`로 제한한다. 에드가는 반필수 최소 대면 때문에 체감 이벤트가 1개 더 많을 수 있다.

### [S3] J4 정보량 조절이 필요하다

#### 문제

사용인 이벤트를 거의 보지 않은 플레이어도 J4를 통해 코어 접근 단서를 얻어야 한다. 하지만 이 경우 정보가 너무 적으면 후반 이해가 약하고, 너무 많으면 관계 이벤트 보상이 줄어든다.

#### 권장안

J4를 `기본`, `확장`, `완전` 3단계로 나눈다.

#### 채택 결과

채택 완료. J4는 `기본 / 확장 / 완전` 3단계 정보량으로 설계하고, 메인 진행에는 기본 복원만 필수로 둔다. 연구원 기록 수는 정보량과 감정 밀도를 바꾸는 변수로 사용한다.

### [S4] 현실 엔딩의 감정적 명분이 더 필요하다

#### 문제

현실이 황폐하다는 사실만으로는 현실 엔딩이 충분히 매력적이지 않을 수 있다.

#### 권장안

현실 엔딩의 핵심 가치는 `나아가는 용기`가 아니라 `거짓이 아닌 시간을 선택하는 것`으로 둔다. 외부 세계의 위험은 온실, 루카의 생체 기록, 거울 진단 패널, J5 기록을 통해 조금씩 누수한다.

#### 채택 결과

채택 완료. v0.3에서는 ED_REALITY의 감정 가치를 `낙관적 탈출`이 아니라 `거짓이 아닌 시간을 선택하는 것`으로 정의한다.

### [S5] 잔류 엔딩이 단순 배드엔딩처럼 고정될 위험이 있다

#### 문제

잔류 엔딩이 단순히 현실 도피로만 보이면 선택의 무게가 약해진다.

#### 권장안

잔류 엔딩은 `거짓임을 알면서도 안정과 애착을 택하는 선택`으로 둔다. 관계도가 높을수록 따뜻하지만 더 서늘하게, 관계도가 낮을수록 감금과 통제의 감각이 강하게 느껴지도록 한다.

#### 채택 결과

채택 완료. v0.3에서는 ED_STAY를 단순 배드엔딩으로 고정하지 않고, 관계 단계에 따라 감금, 공모, 애착의 온도가 달라지는 엔딩으로 설계한다.

### 추가 확인

현재 기준으로 v0.3 본문 작성을 막는 추가 부족분은 없다.

다만 v0.3 이후 상세화 단계에서는 아래 항목을 후속 작업으로 남긴다.

| 항목 | 후속 작업 |
| --- | --- |
| 이벤트 대사 | 실제 게임 대사 단위로 확장 |
| 퍼즐 상세 | 입력 방식, 실패 조건, 힌트 단계 구체화 |
| 오브젝트 반응 | 방별 조사 문장과 루프별 변화 작성 |
| UI | 수첩, 일지, 관계 피드백 표시 방식 결정 |
| 엔딩 콘티 | 화면 구성, 카메라, 사운드, 마지막 컷 작성 |

## 5. 통과 판단

v0.3 작성을 위한 큰 방향은 통과로 판단한다.

따라서 아래 최종 권장 작업을 순차 진행한다.

1. 사용인별 상세 이벤트 목록 작성.
2. J4 기본/확장/완전 복원 문장 작성.
3. 에드가 최소 대면과 전체 에드가 관계 이벤트 분리 작성.
4. E5 저녁 변형표와 F2 연구원 대면 변형표 작성.
5. D5 이후 잠들기 규칙을 상태표로 작성.
6. 엔딩 내부 변형표 작성.
7. 기획서 v0.3 초안 작성.
