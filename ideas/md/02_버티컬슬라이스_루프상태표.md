# GGB 버티컬 슬라이스 루프 상태표

## 1. 목적

이 문서는 시나리오 버티컬 슬라이스에서 루프마다 무엇이 초기화되고 무엇이 유지되는지 정의한다. 추후 Godot에서 이벤트 데이터와 저장 상태를 설계할 때 기준으로 사용할 수 있다.

리셋은 체크포인트 복귀가 아니다. 주인공이 잠들면 언제나 같은 침실의 같은 아침으로 돌아간다. 반복 범위는 시작점을 옮겨 줄이는 것이 아니라, 영구 정보와 숏컷 권한으로 이미 해결한 행동을 빠르게 통과하는 방식으로 줄인다.

## 2. 상태 분류

| 분류 | 설명 | 예시 |
| --- | --- | --- |
| 영구 상태 | 루프를 넘어 유지되는 진행 정보 | 수첩 기록, 루프 인지, 퍼즐 지식, 일지 단계, 숏컷 권한 |
| 루프 상태 | 밤 또는 강제 종료 시 초기화되는 상태 | 일반 아이템, 창문 청소 상태, NPC 위치 |
| 챕터 상태 | 특정 핵심 사건 이후 초기 기준 자체가 변함 | 거울 노출, 균열 단계 |
| 연출 상태 | 진행에는 영향이 적지만 반응성을 만드는 상태 | 어떤 아버지 질문을 선택했는지, 에드가에게 루프를 말했는지 |

## 4. 루프 초기화 규칙

### 리셋 시 유지

- `loop_index`
- `chapter_id`
- `glitch_stage`
- 모든 지식 및 영구 진행 변수
- 모든 이벤트 완료 변수. 단, 이는 숏컷과 대사 판정용이며 물리 상태가 유지된다는 뜻은 아니다.
- 모든 연출 반응 변수
- 수첩 기록
- 해금된 대화 주제
- 플레이어가 직접 확인한 오브젝트 비교 문장
- 해결한 일과의 숏컷 권한
- 실패한 시계와 거울 시도의 원인 기록

### 리셋 시 초기화

- 일반 인벤토리 아이템
- 창문 청소 상태
- 책 배치 상태
- 차 준비 상태
- 세정제 조합 중간 상태
- 완성한 중성 세정제 아이템
- 제거한 시계 봉인핀과 잠긴 시계망
- 굳거나 손상된 거울 코팅
- 문 개폐 상태
- NPC 현재 위치
- 해당 루프에서 소비한 자유 조사 이벤트
- 시간대

### 조건부 유지

| 상태 | 규칙 |
| --- | --- |
| 아버지 일지 | 책장 위치로 돌아가지만 해금된 문장은 유지된다. |
| 열세 번째 종 파형 | 시계 장치 상태는 초기화되지만 성공 시 수첩에 기록된 파형과 일지 반응은 유지된다. |
| 중성 세정제 제조법 | 아이템은 사라지지만 제조법은 유지된다. `L2-01` 완료 후 다음 루프에서는 빠른 제작 가능. |
| 검은 거울 | `L2-02` 이전에는 매 루프 검은 막이 복원된다. `L2-02` 이후에는 챕터 상태로 영구 노출된다. |

### 리셋 후 반복 축소 원칙

| 해결한 내용 | 다음 루프에서 열리는 숏컷 |
| --- | --- |
| 창문 닦기 | 마라에게 보고 후 짧은 몽타주로 완료 |
| 책 정리 | 올바른 책 위치를 알고 있어 한 번의 상호작용으로 완료 |
| 차 준비 | 재료를 먼저 준비해 대화와 조합을 축약 |
| 시간표 조사 | NPC를 다시 추적하지 않고 적절한 시간대에 행동 |
| 일지 복원 | 복원 문장이 처음부터 표시됨 |
| 세정제 제조법 | 재료 확보 후 빠른 제조 가능 |

모든 숏컷은 같은 아침에서 시작한 뒤 실행한다. 플레이어를 중간 사건 위치에 직접 배치하지 않는다.

## 5. 루프별 월드 상태

### Loop 0: 정상적인 하루

| 요소 | 상태 |
| --- | --- |
| 시간대 흐름 | 아침 일과 완료 → 낮 차 준비 → 저녁 자유 조사 → 밤 취침 |
| 에드가 | 침실 → 서재 → 주방 → 복도 → 침실 앞 |
| 마라 | 대응접실 중심 |
| 루카 | 주방 고정 |
| 이리스 | 저녁에만 온실 너머 등장 |
| 일지 | 발견 가능, 읽을 수 없음 |
| 거울 | 평범한 검은 거울. 일반 조사 문장만 제공 |
| 균열 | 시각적 균열 없음. 감각적 암시만 존재 |

### Loop 1: 반복 의심

| 요소 | 상태 |
| --- | --- |
| 주인공 독백 | 프롤로그와 같은 오브젝트에서 비교 문장 추가 |
| 에드가 | 반복 질문을 꿈으로 설명 |
| 잡일 | 다시 수행해야 하지만 일부 대사가 축약됨 |
| 수첩 | 전날 기록 유지. 표시 실험 가능 |
| 일지 | 여전히 읽을 수 없음 |
| 균열 | 새의 반복, 잠금음 등 프롤로그 암시가 더 분명해짐 |

### Loop 2: 반복 확신

| 요소 | 상태 |
| --- | --- |
| 주인공 독백 | 두려움보다 검증과 관찰 중심 |
| 잡일 | 빠르게 돕기 기능 해금 |
| 에드가 | 주인공이 자신을 관찰하면 대화를 먼저 끝냄 |
| 일지 | 저녁에 첫 문장 복원 가능 |
| 서재 | 에드가 시간표를 이용해 제한 조사 가능 |
| 균열 | 멈춘 시계 진동, 동일 음성 반복이 강조됨 |

### Loop 3: 금지된 거울

| 요소 | 상태 |
| --- | --- |
| 주인공 독백 | 금기를 어기는 죄책감과 확인 욕구 중심 |
| 에드가 | 거울 접근 시 즉시 등장. 감시자로서의 성격 강화 |
| 마라 | 세정제 퍼즐의 간접 협조자 |
| 루카 | 중성 용액 제조 힌트 제공 |
| 일지 | 두 번째 문장 복원. 거울 목표 제시 |
| 거울 | 열세 번째 종 이후에만 위장 코팅 제거 가능 |
| 균열 | 거울 노출 직후 단계 `2`로 상승 |

## 6. 시간대별 NPC 위치표

| 인물 | 아침 | 낮 | 저녁 | 밤 |
| --- | --- | --- | --- | --- |
| 에드가 | 침실 / 서재 | 주방 또는 서쪽 복도 | 대시계 확인 후 복도 | 침실로 가는 길 차단 |
| 마라 | 대응접실 | 청소도구실 / 대응접실 | 사용인 구역 방향 | 등장하지 않음 |
| 루카 | 주방 | 주방 | 식당 정리 | 등장하지 않음 |
| 이리스 | 온실 내부 | 온실 내부 | 온실 문 너머 대화 가능 | 등장하지 않음 |

위치표는 기본값이다. 이벤트 진행을 위해 NPC가 이동할 경우 해당 이벤트가 위치표보다 우선한다.

## 9. Godot 데이터화 참고

이번 작업에서는 구현하지 않지만, 추후 아래와 같은 데이터 분리를 권장한다.

### 이벤트 데이터

```yaml
id: L2-02
required_flags:
  neutral_cleaner_ready: true
  thirteenth_bell_known: true
required_time_phase: evening
set_flags:
  mirror_revealed: true
  basement_coordinates_known: true
  glitch_stage: 2
next_objective: find_basement_entrance
```

### 오브젝트 반응 우선순위

1. 현재 이벤트 전용 반응.
2. 챕터 또는 균열 단계 반응.
3. 루프 인지 단계 반응.
4. 기본 반응.

### 저장 권장 단위

- 영구 진행 상태는 세이브 데이터에 저장한다.
- 루프 상태는 현재 루프 세션에 저장하고 리셋 시 초기화한다.
- 수첩 기록은 배열보다 기록 ID 집합으로 저장해 중복을 방지한다.
- 이벤트 완료 플래그와 지식 플래그를 분리해, 이벤트를 보지 않았더라도 다른 경로로 정보를 얻는 확장을 허용한다.
